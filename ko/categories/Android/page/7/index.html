[안드로이드] URI와 UriMatcher

2019/01/19 카테고리 Android

Android의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

URI

URI(Uniform Resource Identifier, 통합 자원 식별자)는 자원을 구분해주는 식별자입니다. 내 스마트폰이나 데스크톱, 그리고 인터넷 상에는 무수히 많은 자원(파일, 이미지, 비디오 등)이 존재합니다. URI는 이 자원을 구분하고 식별하는데 사용됩니다.

우리는 구글에 접속할 때 웹 브라우저의 주소창에 http://www.google.com를 입력합니다. 이때 http://www.google.com 뒤에는 index.html이 생략되어있습니다

http://www.google.com/index.html

이 주소도 URI입니다. 웹 브라우저는 URI에 위치한 index.html 파일을 가져와서 화면에 보여줍니다. 이처럼 URI는 자원을 구분하고 식별하는데 사용됩니다.

URI 구조

URI는 다음과 같은 구조를 가지고 있습니다.

scheme와 host는 필수사항이고 나머지는 선택사항입니다.

(2) Authority

authority는 사용자 이름과 비밀번호, 호스트, 포트번호의 조합입니다.

(3) Host

네트워크에 연결되어 있는 모든 컴퓨터 각각을 호스트라고 합니다. 보통 도메인 네임이나 IP 주소을 사용하여 호스트를 구분하고 식별합니다.

(4) Port

컴퓨터 내에서 실행되고있는 어플리케이션들을 구분하기 위해 사용하는 번호입니다. 보통 HTTP 프로토콜을 사용할 때는 80, HTTPS 프로토콜을 사용할 때는 443 포트를 사용합니다. 예제는 다음과 같습니다.

[안드로이드] compileSdkVersion, minSdkVersion, targetSdkVersion 차이점

2019/01/16 카테고리 Android

Android의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 런타임에서 SDK 버전을 확인하는 방법에 대해 알아봤습니다. 이번 포스트에서는 compileSdkVersion, minSdkVersion, targetSdkVersion의 차이점에 대해 알아보겠습니다.

(1) compileSdkVersion

안드로이드 스튜디오에서 앱을 빌드할 때 어떤 SDK 버전을 사용할 것인지 지정합니다. 즉 comileSdkVersion은 지원할 수 있는 가장 높은 버전을 의미합니다.

안드로이드 스튜디오의 SDK Manager에 해당 SDK가 설치되어있어야 합니다.

(2) minSdkVersion

최소 지원 SDK 버전을 의미합니다. 즉 해당 버전 이하의 안드로이드 운영체제가 설치된 기기에서는 앱을 설치할 수 없습니다.

(3) targetSdkVersion

앱을 실제 기기에 설치하고 실행할 때 어떤 SDK 버전을 사용할지 지정합니다. 기능은 똑같아도 SDK 버전에 따라 다르게 동작하는 API도 있습니다. 이때는 targetSdkVersion 값에 맞는 API를 사용하게 됩니다.

targetSdkVersion을 지정하지 않으면 기본 값은 minSdkVersion과 동일하게 적용됩니다.

예제

모듈 수준의 build.gradle파일에서 다음과 같이 SDK 버전 설정을 설정합니다.

이는 SDK 버전이 25 이상 30 이하에서만 이 앱이 실행될 수 있음을 의미합니다. 또한 실제 동작은 27 버전으로 동작하게 됩니다. 보통 실무에서는

minSdkVersion <= targetSdkVersion == compileSdkVersion

로 SDK 버전을 설정합니다.

마치며

이번 포스트에서는 compileSdkVersion, minSdkVersion, targetSdkVersion 차이점에 대해 알아봤습니다. 다음 포스트에서는 ADB(Android Debug Bridge)에 대해 알아보겠습니다.

[안드로이드] 안드로이드 SDK란?

2019/01/15 카테고리 Android

Android의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 구글 플레이 스토어에 앱을 출시하는 방법에 대해 알아봤습니다. 이번 포스트에서는 안드로이드 SDK에 대해 알아보겠습니다.

안드로이드 SDK

안드로이드 SDK(Software Development Kit)는 안드로이드 앱을 개발하기 위한 툴 킷입니다.

안드로이드 스튜디오에서는 SDK Manager를 제공하는데 이 곳에서 안드로이드 SDK의 설치나 삭제, 설정을 할 수 있습니다. 또한 SDK Manager에서 SDK의 설치 경로를 확인할 수 있습니다.

그럼 터미널에서 SDK의 설치 경로로 이동해보겠습니다.

SDK에 무엇이 설치되어있는지 확인해봅시다.

(1) system-images

system-images폴더에는 에뮬레이터에서 사용되는 안드로이드 운영체제 이미지가 있습니다.

(2) platforms

platforms폴더에는 안드로이드 플랫폼이 설치되어 있습니다.

(3) tools

tools폴더와 tools/bin폴더에는 앱을 개발하고 디버깅하기 위한 여러가지 툴이 있습니다. 에뮬레이터, SD 매니저, AVD 매니저, UI 뷰어 등이 이 폴더에 설치되어있습니다.

안드로이드 스튜디오에서도 tools폴더에 설치된 툴을 확인할 수 있습니다.

(4) build-tools

build-tools폴더에는 앱을 빌드하기 위한 도구들이 설치되어있습니다.

(5) platform-tools

platform-tools폴더에는 안드로이드 기기와 연결했을 때 사용하는 도구들이 설치되어있습니다. adb, fastboot, systrace, sqlite3 등이 이 폴더에 설치되어있습니다.

마치며

이번 포스트에서는 안드로이드 SDK에 대해 알아봤습니다. 다음 포스트에서는 런타임에서 SDK 버전을 확인하는 방법에 대해 알아보겠습니다.

[안드로이드] 앱 출시하기 - (1) 서명된 릴리즈 APK 생성

2018/12/23 카테고리 Android

Android의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

앞으로 3개의 포스트에 걸쳐 앱을 출시하는 방법에 대해 알아보겠습니다.

  1. 서명된 릴리즈 APK 생성 ⬅️ ⬅️ ⬅️
  2. Google Play Store에 출시하기
  3. 새로운 버전 출시하기

앱 출시 프로세스

구글 플레이 스토어에 앱을 출시할 때는 다음 과정을 거칩니다.

  1. 디버그 모드 대신 릴리스 모드로 변경
  2. 앱 최적화와 역공학 방지를 위한 추가 작업
  3. 개인 키를 사용하여 앱에 서명하고 빌드
  4. 구글 플레이 스토어에 서명된 APK 파일 업로드

1. 디버그 모드

개발 단계에서는 디버그 모드로 앱을 빌드하고 실행합니다.

안드로이드 스튜디오는 디버그 모드일 때 디버그용 키를 사용하여 앱을 자동으로 서명합니다. 따라서 에뮬레이터나 연결된 실제 기기에 앱을 설치하고 실행할 때 별도의 서명 정보 입력이 필요없습니다.

안드로이드 스튜디오는 프로젝트를 처음 실행하면 디버그용 키를 자동으로 생성합니다. 이 키가 저장된 키 스토어의 위치는 $HOME/.android/debug.keystore입니다.

디버그용 키는 설계상 보안이 유지되지 않습니다. 따라서 구글 플레이 스토어를 포함한 대부분의 앱 스토어는 디버그용 키로 서명한 앱을 게시하는 것을 허용하지 않습니다.

2. 릴리즈 모드

플레이 스토어에 앱을 출시하려면 서명된 APK 파일을 업로드 해야합니다. 서명된 APK 파일을 생성하려면 빌드 타입(BuildType)를 릴리즈 모드로 변경해야합니다.

3. 키스토어 및 키 생성

소스코드와 리소스 파일을 빌드하고 개인 키로 서명하면 서명된 APK 파일이 생성됩니다. 이 개인 키가 저장되는 곳을 키 스토어(Key Store)라고 합니다.

우선 키 스토어를 생성합시다. Build > Generate Signed APK 메뉴로 들어갑니다.

대화 상자에서 APK를 선택합니다.

먼저 키 스토어를 생성합시다. Create new버튼을 클릭합니다. 만약 이전에 키 스토어를 만들었다면 Choose existing버튼을 클릭하여 기존의 키 스토어를 사용합니다.

New Key Store 대화상자는 두 개의 색션으로 나뉘어 있습니다. 첫 번째 색션은 키 스토어 파일과 관련있습니다. 키 스토어 파일의 위치와 비밀번호를 입력합니다.

두 번째 색션은 키 스토어에 저장될 개인 키와 관련있습니다. 별칭(Alias), 비밀번호, 유효기간은 필수로 입력해야합니다. 나머지 필드는 최소 하나의 필드에 정보를 입력합니다.

간혹 다음과 같은 경고 문구가 발생합니다. jks 키 저장소의 형식의 사용을 권장하는 경고문인데 APK 파일에 서명하는데 전혀 지장없으므로 무시하셔도 됩니다.

정보가 제대로 입력되었는지 확인하고 Next버튼을 누릅니다.

Destination Folder에서는 APK 파일이 생성될 경로를 지정합니다. 기본값은 프로젝트의 app 폴더입니다. Build Variants는 release를 선택합니다. Signature Version은 V2를 선택하고 Finish 버튼을 누릅니다.

이제 app 폴더를 확인해봅시다. [your_project]/app/release 폴더 안에 서명된 APK 파일이 생성되었습니다.

[안드로이드] APK 파일이란?

2018/12/18 카테고리 Android

Android의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

APK

APK(Android Application Package)는 안드로이드 설치 파일입니다. 안드로이드 시스템에서 APK 파일을 실행하여 어플리케이션을 설치할 수 있습니다. Windows 계열의 운영체제에서 installer 역할을 한다고 보시면 됩니다.

빌드 프로세스

APK 파일은 다음과 같은 과정을 거쳐 생성됩니다.

  1. 안드로이드 프로젝트는 소스코드 및 리소스로 구성된 모듈과 의존성으로 구성됩니다.
  2. 컴파일러는 이 모든 것을 빌드하여 DEX 파일(Dalvik Executable File)과 컴파일된 리소스를 생성합니다.
  3. APK Manager 개인키로 서명하여 APK 파일를 생성합니다.
  4. APK 파일은 플레이스토어에 업로드하거나 안드로이드 시스템에 직접 설치할 수 있습니다.

APK 위치

디버그 모드에서 앱을 빌드하면 아래 경로에 APK 파일이 생성됩니다
