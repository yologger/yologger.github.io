[Android] 모듈(Module)이란?

2019/01/04 카테고리 Android, Gradle

개념

Android Developers에서는 모듈(Module)을 다음과 같이 정의하고 있습니다.

모듈은 소스 파일 및 빌드 설정으로 구성된 모음이며, 이를 통해 프로젝트를 별개의 기능 단위로 분할할 수 있습니다. 프로젝트에는 하나 이상의 모듈이 포함될 수 있으며, 하나의 모듈이 다른 모듈을 종속 항목으로 사용할 수 있습니다. 각 모듈은 개별적으로 빌드, 테스트 및 디버그할 수 있습니다.

프로젝트와 모듈을 직접 만들어보면서 모듈이 무엇인지 예제를 통해 알아보겠습니다.

모듈 생성하기

일단 안드로이드 스튜디오에서 프로젝트를 생성하면 app이라는 모듈이 자동으로 생성됩니다.

이제 새로운 모듈을 추가해보겠습니다. File > New > New Module을 클릭하면 생성할 모듈을 선택할 수 있습니다. 일단 Java or Kotlin Module을 선택합시다.

모듈의 이름은 test_module입니다.

모듈 사용하기

이제 모듈 app에서 모듈 test_module의 TestModule클래스를 사용하고자 합니다. 이때는 모듈 app의 build.gradle에 의존성을 추가해야합니다.

build.gradle(Module: my_project.app)

이제 모듈 app에서 TestModule을 사용할 수 있습니다.

멀티 모듈의 장점

한 프로젝트 내에서 여러 개의 모듈을 사용하는 것을 멀티 모듈이라고 합니다.

위 예제를 보면 모듈 app에 대한 build.gradle파일, 모듈 test_module에 대한 build.gradle파일이 각각 존재하는 걸 알 수 있습니다. 이는 각 모듈을 별도로 빌드할 수 있다는 뜻입니다. 이를 통해 빌드 시간을 단축시킬 수 있습니다.

또한 기능별로 코드를 분리하여 중복을 제거하고 재사용성을 높일 수 있습니다. 뿐만 아니라 모듈을 각각 테스트를 할 수도 있어 테스트 용이성이 높아집니다.

모듈의 종류

자주 사용하는 모듈에는 크게 3가지 종류가 있습니다.

1. Phone & Tablet Module

빌드 결과로 안드로이드 운영체제에서 실행 가능한 .apk 파일을 생성합니다. 똑같은 앱을 스마트폰, 태플릿 용도로 나누어 출시한다던가, 유료/무료 앱을 구분하여 출시할 때 사용할 수 있습니다. 프로젝트를 생성하면 기본으로 생성되는 app모듈이 바로 Phone & Tablet Module입니다.

의문점

안드로이드 스튜디오를 사용하는 개발자는 한 가지 의문이 생길 수도 있습니다. 그레이들을 별도로 설치하지 않았는데도 프로젝트를 빌드할 수 있기 때문입니다. 이를 가능하게 하는 것이 바로 그레이들 래퍼(Gradle Wrapper)입니다.

Gradle Wrapper

그래이들 래퍼(Gradle Wrapper)는 이미 존재하는 프로젝트를 새로운 환경에 설치할 때, 별도의 그래이들 설치 과정없이 바로 빌드할 수 있도록 프로젝트 내부에 빌드 스크립트 파일을 포함시키는 방법입니다.

우리는 이전 포스트에서 그래이들을 설치하고 아래와 같은 명령어를 입력하여 일(task)를 수행했습니다.

하지만 위 처럼 그레이들 명령어를 사용하려면 빌드하는 PC에 그레이들이 설치되어있어야 합니다. 이제 그래이틀 래퍼를 사용하여 프로젝트를 빌드해보겠습니다.

이제 디렉토리를 확인해보면 그레이들, 그레이들 래퍼와 관련된 파일이 생성된 것을 확인할 수 있습니다.

Gradle Wrapper 구조

생성된 디렉토리의 구조는 다음과 같습니다.

1. gradlew.bat

윈도우용 wrapper 실행 스크립트 파일입니다.

2. gradlew

유닉스용 wrapper 실행 스크립트 파일입니다.

3. gradle/wrapper/gradle-wrapper.jar

wrapper 파일입니다. 위 2개의 스크립트 파일 중 하나가 실행되면 운영체제에 맞는 환경을 로컬 캐시에 다운받은 뒤 실제 일(task)를 실행합니다.

4. gradle/wrapper/gradle-wrapper.properties

wrapper 설정파일입니다.

[Android] 컴파일(Compile)과 빌드(Build)의 차이점

2019/01/01 카테고리 Android, Gradle

컴파일(Compile)

컴파일(Compile)은 개발자가 작성한 소스코드를 컴퓨터가 이해할 수 있는 기계어로 변환하는 과정을 의미합니다.

C의 경우 gcc라는 컴파일러로 *.c파일을 컴파일하며, *.o라는 파일을 반환합니다. Java의 경우 javac(Java Compiler)라는 컴파일러로 *.java파일을 컴파일하며 자바 바이트코드라는 결과물을 반환합니다.

빌드(Build)

프로젝트를 진행하다보면 여러 개의 소스코드 파일이 생성되고, A라는 소스코드 파일에서 B라는 소스코드 파일을 참조하는 경우가 있습니다. 이때 파일 A와 파일 B를 각각 컴파일하면 A에서 B를 찾지 못하므로 참조할 수가 없습니다. 따라서 파일 A와 파일 B를 연결해주는 작업이 필요한데 이를 링크(Link)라고 합니다.

빌드(Build)는 이러한 컴파일, 링크 작업 등을 모두 포함하여 실행 가능한 소프트웨어 결과물로 만드는 과정을 의미합니다.

안드로이드 스튜디오에서 소스코드를 빌드하면 안드로이드 운영체제에서 실행할 수 있는 apk(Android Package)가 생성됩니다.

빌드 도구(Build Tool)

빌드 도구(Build Tool)은 말 그대로 소스코드를 실행 가능한 소프트웨어로 만들어주는 도구입니다. 최근 빌드 툴은 빌드 작업뿐만 아니라 테스트, 빌드 자동화, 배포 기능까지 제공하기도 합니다. Java나 Kotlin의 빌드 툴에는 Ant, Maven, Gradle 등이 있습니다.
