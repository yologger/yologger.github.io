[Kotlin] 접근 제한자(Access Modifiers)

2019/02/06 카테고리 Kotlin

Kotlin의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

접근 제한자 (Access Modifiers)

접근 제한자는 외부에서 클래스 내부의 메소드나 멤버변수에 접근하는 것을 제한하는 것입니다. Kotlin은 private, protected, internal, public이라는 4개의 접근 제한자를 지원합니다.

private

private으로 선언된 변수는 해당 클래스 내부에서만 접근이 가능합니다.

private으로 선언된 변수는 상속받는 자식 클래스에서도 접근이 불가능합니다.

protected

protected로 선언된 변수는 해당 클래스와 자식 클래스에서 접근이 가능합니다.

internal

internal로 선언된 변수는 같은 모듈 내에서는 어디서 접근이 가능합니다. 코틀린 공식 문서에서 말하는 같은 모듈은 아래 상황을 뜻합니다.

  1. IntelliJ IDEA Module
  2. Maven Project
  3. Gradle Source Set(with the exception that the test source set can access the internal declarations of main)
  4. a set of files compiled with one invocation of the Ant task

쉽게 말하면 같은 프로젝트 내에서는 제한자 internal에 접근 가능하다고 보면 되겠습니다.

public (default)

어디서든 접근이 가능합니다. 접근 제한자를 붙이지 않으면 기본적으로 public입니다.

[Kotlin] 1급 객체, 1급 함수와 함수형 프로그래밍

2019/02/03 카테고리 Kotlin

Kotlin의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 람다식과 익명함수에 대해 알아봤습니다. 이번 포스트에서는 1급 객체와 1급 함수, 그리고 함수형 프로그래밍에 대해 알아보겠습니다.

1급 객체(First-class Object)

다음 3가지 조건을 모두 충족하면 1급 객체라고 힙니다.

  1. 변수나 상수에 할당할 수 있다.
  2. 함수의 인자로 전달할 수 있다.
  3. 함수의 반환값으로 반환할 수 있다.

Kotlin에서는 함수를 1급 객체로 취급합니다. 함수가 위 3가지 조건을 모두 충족하는지 살펴보겠습니다.

1. 함수를 변수나 상수에 할당할 수 있다.

Kotlin에서는 함수를 변수에 할당할 수 있습니다. 이때 리플렉션(::)을 사용합니다.

익명함수 또한 변수에 할당할 수 있습니다.

2. 함수를 함수의 인자로 전달할 수 있다.

Kotlin에서는 함수를 함수의 인자로 전달할 수 있습니다.

3. 함수를 함수의 반환 값으로 반환

Kotlin에서는 함수를 함수의 반환 값으로 반환할 수 있습니다.

함수 타입

함수 타입은 위 3가지 특성을 가능하게 해주는 Kotlin의 자료형입니다. 정수형 타입의 변수에 정수를 할당하는 것 처럼 함수형 타입에는 함수를 할당할 수 있습니다.

위 예제에서는 (Int, Int) -> Int가 바로 함수 타입입니다.

마치며

이번 포스트에서는 1급 객체와 1급 함수, 그리고 함수형 프로그래밍에 대해 알아봤습니다. 다음 포스트에서는 Unit에 대해 알아보겠습니다.

[Kotlin] 에러 처리(Exception & Error Handling)

2019/02/01 카테고리 Kotlin

Kotlin의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이번 포스트에서는 Kotlin의 에러 처리에 대해 알아보겠습니다.

Throwable

Kotlin은 오류가 발생하면 Error나 Exception을 발생시킵니다. Error와 Exception은 모두 Throwable클래스를 상속받습니다.

Error

Error는 코드에서 잡아낼 수 없으며 어플리케이션 자체가 다운됩니다. 대표적으로 OutOfMemoryError, StackOveflowError가 있습니다.

Exception

Exception은 try-catch구문으로 잡아낼 수 있으며 프로그램이 다운되지 않도록 처리할 수 있습니다. 대표적으로 값이 null인 변수에 접근하면 발생하는 NullPointException이 있습니다.

개발자가 직접 Exception을 정의하고 발생시킬 수도 있습니다.

직접 Exception을 발생시킬 때는 키워드 throw를 사용합니다.

Exception을 발생시킬 수 있는 메소드에는 @Throws 어노테이션을 붙여 에러 가능성을 표기할 수 있습니다.

@Throws어노테이션이 붙은 메소드는 try-catch문 내에서 호출하는 걸 권장합니다.

마치며

[Kotlin] 상수(val, const)

2019/01/28 카테고리 Kotlin

Kotlin의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

우리는 이전 포스트에서 스코프 함수에 대해 알아봤습니다. 이번 포스트에서는 상수 (val, const)에 대해 알아보겠습니다.

변수 var

변수를 선언할 때는 키워드 var를 사용합니다. 변수는 런타임에서 값을 수시로 변경할 수 있습니다.

상수 val

val은 런타임에 값이 결정되는 상수입니다. 상수는 한 번 값이 할당되면 변경할 수 없습니다.

val은 문자열도 할당할 수 있습니다.

val는 클래스의 인스턴스도 할당할 수 있습니다.

const val

const val는 컴파일 타임에 값이 결정되는 상수입니다. 인스턴스를 할당할 수 없으며, 기본 자료형과 문자열만 할당할 수 있습니다. const val상수의 이름은 통상 대문자와 언더바를 통해 선언합니다.

클래스 안에서 사용할 때는 companion object 블럭 안에 정의합니다.

마치며

이번 포스트에서는 상수(val, const)에 대해 알아봤습니다. 다음 포스트에서는 스코프 함수 중 늦은 초기화의 lateinit에 대해 알아보겠습니다.
