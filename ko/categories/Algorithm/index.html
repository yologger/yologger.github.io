이번 포스트에서는 자바 컴파일러, JVM, JRE, JDK의 차이점에 대해 알아보겠습니다.

자바 컴파일러

우리가 자바로 작성한 파일명 뒤에는 .java라는 확장자가 붙습니다. 이를 보통 자바 소스코드(Java Source Code)라고 합니다. 자바 컴파일러(Java Compiler)는 이 소스코드를 자바 바이트코드(Java Bytecode)로 변환합니다. 자바 바이트코드는 파일명 뒤에 .class확장자가 붙습니다.

JVM

JVM은 Java Virtual Machine의 줄임말로 자바 바이트코드(Java Bytecode)를 운영체제에 종속적인 코드로 변환하여 실제로 실행합니다.

아래 그림처럼 서로 다른 운영체제라도 JVM만 설치되어 있다면, 같은 자바 어플리케이션을 다른 운영체제에서 실행할 수 있습니다.


단, 자바 어플리케이션과는 달리 JVM은 운영체제에 종속적이므로 각 운영체제에 맞는 JVM을 설치해야 합니다.

JRE

JRE는 Java Runtime Edition의 줄임말로 Java언어로 작성된 애플리케이션을 실행하는데 필요한 요소들입니다. JVM과 실행에 필요한 Java API로 구성되어 있습니다.

JDK

JDK는 Java Development Kit의 줄임말로 어플리케이션 개발에 필요한 요소들입니다. JDK는 JRE 외에도 Java Compiler, Java Debugger 등 개발에 필요한 다양한 도구를 포함하고 있습니다.

Java 7, Java SE 7, JDK 7의 차이점

Java SE는 Java Platform Standard Edition의 약자로, Java Platform의 근간을 이루는 명세(Specification)를 가리킵니다. JDK는 명세를 코드로 구현한 구현체(Implementation)입니다. 즉 Java SE 7은 JSR-336로 규정된 Java SE의 일곱 번째 명세를 의미하며, JDK 7은 OpenJDK JDK 7, Oracle JDK 7과 같은 구현체를 의미합니다.

추가적으로 Java 7은 정확한 표현이 아닙니다. 하지만 Java SE 7이나 JDK 7보다 사용하기 편해서 흔히 Java 7이라고 부르는 것입니다.
