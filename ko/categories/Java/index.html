[Java] 자바 유료화? Oracle JDK와 OpenJDK

2018/04/03 카테고리 Java

자바(Java)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

2018년 Oracle이 Java SE의 과금 방식을 영구 라이선스에서 구독 모델로 변경하겠다고 밝혔습니다. 이 선언이 그동안 무료로 인식된 Java의 유료화로 받아들여지면서 관련 업계에 큰 혼란을 초래했습니다. Java의 유료화를 이해하려면 Oracle JDK와 OpenJDK에 대해 알아야 합니다.

참고로 Java 언어 자체는 무료입니다. Java 언어로 작성된 어플리케이션을 컴파일하고 실행하는 JDK(Java Development Kit)가 유료화 논쟁의 대상입니다.

Oracle JDK

Oracle JDK는 사용 목적에 따라 유료이기도 무료이기도 한 JDK입니다.

서버, PC, 스마트폰 등의 일반적인 목적으로 사용될 때는 무료이지만 키오스크 등 특수한 목적의 컴퓨팅에 사용될 땐 유료입니다. 또한 Oracle JDK 8(Oracle Java SE 8)부터는 Oracle의 지속적인 업데이트와 보안 패치, 기술지원을 받으려면 유료 서비스를 구매해야합니다. (참고로 Oracle JDK 8 이전 버전에서도 일반적인 목적의 사용은 무료였고 지속적인 업데이트와 보안 패치, 기술지원은 유료였습니다.)

Oracle JDK는 OpenJDK를 기본으로 합니다. Open JDK에 추가적인 기능이 더해진 것이 Oracle JDK입니다. Oracle JDK에 존재하고 Open JDK에는 없는 대표적 기능으로는 글꼴 라이브러리와 Java Web Start가 있습니다.

OpenJDK

OpenJDK는 사용 목적에 관계없이 무료인 JDK입니다.

2007년 Java를 개발한 Sun Microsystems사는 JDK를 오픈소스화하기 위해 OpenJDK를 만들었습니다. Sun Microsystems사는 저작권자가 오픈소스화를 거부한 일부 기능을 제외한 나머지 JDK 소스코드 전부를 OpenJDK에 제공했고, OpenJDK는 이를 기반으로 JDK 7 프로젝트를 시작했습니다.

JDK 6 시절에는 OpenJDK가 Oracle JDK보다 성능이나 안정성이 크게 떨어졌습니다. 하지만 오늘날에는 Oracle JDK만이 제공하는 일부 기능을 제외하고는 차이가 없다는게 Oracle의 설명입니다.

안드로이드 스튜디오의 JDK

참고로 안드로이드 스튜디오에는 OpenJDK가 내장되어 있으므로 별도의 JDK 설치가 필요없습니다. File > Project Structure > SDK Location에서 경로를 확인할 수 있습니다.

안드로이드 스튜디오에서는 보통 compileSDKVersion에 따라 적당한 JDK 버전이 자동으로 설정됩니다. 물론 프로젝트 수준의 build.gradle 파일에서 직접 JDK 버전을 설정할 수 있습니다.

결론

Kakao나 Line처럼 한국에서 잘 알려진 기업도 OpenJDK를 도입하고 활용하고 있는 것으로 알고 있습니다. OpenJDK에서는 제공하지 않는 컴포넌트를 사용해야하거나, Oracle의 지속적인 지원을 받아야 하거나, 키오스크 등 특수한 목적의 컴퓨팅에 사용하는 경우에는 비용을 지불하고 Oracle JDK를 사용해야할 것입니다. 그렇지 않은 경우에는 OpenJDK를 사용해도 무방할 것 같습니다.

[자바] 자바 플랫폼(Java Platform)의 종류

2018/04/02 카테고리 Java

자바(Java)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

자바 플랫폼(Java Platform)에는 여러 에디션이 존재합니다. 이번 포스트에서는 대표적인 세 가지 에디션의 특징에 대해 알아보겠습니다.

JDK에 대해 다루고 있는 이 포스트도 참조하시길 권장합니다.

Java SE (Java Platform, Standard Edition)

  • 과거에는 J2SE(Java 2 SE)로 불렸습니다.
  • 가장 보편적으로 사용되는 에디션입니다.
  • 자바 데스크탑, 스마트폰 어플리케이션 개발에 사용됩니다.
  • 안드로이드 어플리케이션을 개발할 때에도 주로 Java SE가 사용됩니다.
  • 가상머신 외에도 기초적인 타입부터 네트워킹, 보안, 데이터베이스 처리, 그래픽 사용자 인터페이스 개발(Swing, AWT) 등 데스크탑 개발에 필요한 API를 포함합니다.

Java EE (Java Platform, Enterprise Edition)

  • 과거에는 J2EE(Java 2 EE)로 불렸습니다.
  • 자바 서버 애플리케이션 개발에 사용됩니다.
  • Java SE의 모든 API를 포함합니다.
  • 그 외 EJB, JSP, Sevlet, JDBC 등 서버 어플리케이션 개발을 위한 API가 포함됩니다.
  • 국내에서는 Java SE, Spring Framework, Tomcat를 조합한 기술스택에 밀려 거의 사장된 상태입니다.
  • 2017년 오라클은 오픈소스 진영인 이클립스 재단에 Java EE를 넘겼으며, 명칭은 Jakarta EE로 변경되었습니다.

Java ME (Java Platform, Micro Edition)

  • Java SE API의 일부만 포함합니다.
  • 셋톱박스, PDA등 소형 임베디드 시스템을 개발하는데 사용됩니다.


각각의 에디션도 사용하는 운영체제(Window, MacOs, Linux, ..)에 따라 다른 Java Platform을 다운받아야합니다. Java Platform의 모든 에디션은 이 곳에서 다운받을 수 있습니다.

[자바] 람다식(익명함수)와 함수형 인터페이스(Functional Interface)

2018/03/16 카테고리 Java

자바(Java)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 일급 객체, 함수형 프로그래밍에 대해 알아봤습니다. 이번 포스트에서는 람다식(익명함수), 함수형 인터페이스에 대해 알아보겠습니다.

람다식

람다식(Lambda Expression)은 함수를 하나의 식(expression)으로 표현한 것입니다. 함수를 람다식으로 표현하면 메소드의 이름이 필요 없기 때문에 익명 함수(Anonymous Function)라고도 불립니다.

코틀린에서의 람다식

코틀린(Kotlin), 스위프트(Swift)처럼 비교적 최근에 개발된 프로그래밍 언어는 자체적으로 람다식을 지원합니다. 그럼 코틀린 먼저 예제를 살펴보겠습니다. 보통 코틀린에서 함수는 다음과 같이 선언합니다.

코틀린은 함수를 일급 객체로 취급하므로 람다식을 다음 코드와 같이 변수에 저장할 수 있습니다. 이는 언어 자체에서 함수 타입을 지원하기 때문입니다. 정수 타입에는 정수를 할당하는 것처럼 함수 타입에는 함수를 할당할 수 있습니다.

자바에서의 람다식

자바는 코틀린, 스위프트 같은 언어에 비해 비교적 오래되었습니다. 또한 객체지향 프로그래밍을 패러다임으로 만들어졌기 때문에 람다식과 함수형 프로그래밍을 지원하지 않았습니다. 그러나 2014년 발표된 Java 8부터 함수형 프로그래밍을 지원하기 위한 자바 API가 추가되었습니다.

아쉽게도 자바는 함수 타입이 없으므로 람다식을 변수에 저장하거나 매개변수로 전달할 수 없습니다. 이를 해결하기 위해 도입된 개념이 @FunctionalInterface어노테이션 입니다.

@FunctionInterface

함수형 인터페이스를 구현하기 위해서는 인터페이스를 정의하고 그 내부에는 한 개 뿐인 추상 메소드를 선언하고, @FunctionalInterface 어노테이션을 붙여주면 됩니다.

이제 마치 변수에 함수를 할당하는 것처럼 사용할 수 있습니다.

자바 API에서는 자주 사용될 것 같은 함수형 인터페이스가 이미 정의되어있습니다.

(1) Supplier

매개변수 없이 반환값만 갖는 함수형 인터페이스입니다. Supplier는 T get()을 추상 메소드로 갖고 있습니다.

(2) Consumer

매개변수가 있고 반환값은 없는 함수형 인터페이스입니다. Consumer는 void accept(T t)를 추상메소드로 갖고 있습니다.

(3) Function <T, R>

매개변수도 있고 반환값도 있는 함수형 인터페이스입니다. Function은 R apply(T t)를 추상 메소드로 가지고 있습니다.

(4) Predicate

매개변수가 있고 boolean타입의 값을 반환합니다. Supplier는 boolean test(T t)을 추상 메소드로 갖고 있습니다.

안드로이드에서의 람다식

람다식은 안드로이드 어플리케이션 개발에서도 자주 사용합니다. 보통 버튼같은 위젯을 클릭했을 때 실행할 구문을 람다식 안에 구현합니다.

우선 람다식을 사용하지 않고 익명 클래스를 사용하여 특정 버튼을 클릭했을 때 실행할 구문을 등록해보겠습니다.

람다식을 사용하면 다음과 같습니다.

데이터 타입 생략도 가능합니다.

함수의 몸체가 한 줄일 때는 중괄호{}도 생략하며, 한 줄로 표현할 수 있습니다.

매개변수가 한 개일 때는 소괄호()도 생략할 수 있습니다.

마치며

[자바] 일급 객체, 함수형 프로그래밍

2018/03/15 카테고리 Java

자바(Java)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 어노테이션(Annotation)에 대해 알아봤습니다. 이번 포스트에서는 일급 객체, 함수형 프로그래밍에 대해 알아보겠습니다.

일급 객체 (First-class citizen)

함수형 프로그래밍을 공부하다보면 일급 객체란 개념이 많이 언급됩니다. 다음 세 가지 조건을 모두 충족하면 일급 객체라고 할 수 있습니다.

  1. 변수나 상수에 할당할 수 있어야 한다.
  2. 함수의 인자로 전달할 수 있어야 한다.
  3. 함수의 반환값으로 반환할 수 있어야 한다.

코틀린에서 함수는 일급 객체이지만 자바에서 함수는 일급 객체가 아닙니다. 우선 코틀린 예제를 살펴보겠습니다.

위 예제처럼 코틀린은 함수를 변수에 할당할 수 있으므로 함수를 일급 객체로 취급합니다. 그럼 자바 예제를 살펴볼까요?

코틀린은 함수를 함수의 인자로 전달할 수 있고 함수의 반환값으로 반환할 수 있습니다. 반면 자바는 함수를 함수의 인자로 전달할 수 없고 함수의 반환값으로 반환할 수도 없습니다.

함수형 프로그래밍 언어 (Functional Programming)

함수형 프로그래밍 언어는 함수를 일급 객체로 간주하여 변수에도 할당할 수 있고, 함수의 인자로 전달할 수 있으며, 함수의 반환값으로 반환할 수 있는 언어를 의미합니다.

마치며

이번 포스트에서는 일급 객체와 함수형 프로그래밍에 대해 알아봤습니다. 다음 포스트에서는 람다식(익명 함수)과 함수형 인터페이스에 대해 알아보겠습니다.

어노테이션 프로세싱

어노테이션 프로세싱은 자바 컴파일러가 컴파일 단계에서 유저가 정의한 어노테이션을 분석하고 처리하는 기법입니다.

자바 컴파일러와 어노테이션 프로세서

자바 컴파일러(javac)는 자바 소스코드(.java)를 자바 바이트코드(.class)로 변환합니다. 이때 어노테이션 프로세서(Annotation Processor)를 플러그인 형태로 자바 컴파일러에 등록하여 사용합니다.

자바 컴파일러는 등록된 어노테이션 프로세서가 모두 처리될 때까지 반복하여 실행합니다. 이 때 어노테이션 프로세서에 구현된 로직에 따라 경고나 에러를 발생시키기도, 추가적인 자바코드를 추가하기도 합니다. 자바 컴파일러는 이렇게 최종적으로 완성된 자바 소스코드를 자바 바이트코드로 변환합니다.

  • init(): 파일을 생성하기 위해 필요한 FileWriter, 디버깅에 필요한 Messager 등 각종 유틸리티 클래스들을 이 곳에서 사용할 수 있습니다.
  • process(): 프로세서의 핵심 부분입니다. 이곳에서 클래스, 메소드, 필드 등에 추가한 어노테이션을 처리하고 처리에 대한 결과로 자바 파일을 생성할 수 있습니다.
  • getSupportedAnnotationType(): 어떤 어노테이션들을 처리할 지 집합(Set) 형태로 리턴하게 됩니다.
  • getSupportedSourceVersion(): 일반적으로 최신의 자바 버전을 리턴합니다.

어노테이션

자바로 개발을 하다 보면 클래스, 메서드, 변수 앞에 @Override 같은 @ 표시를 많이 봤을 것입니다. @를 어노테이션(Annotation)이라고 합니다.

어노테이션은 컴파일 타임이나 런타임에 특정 처리를 하도록 컴파일러에게 정보를 제공합니다. 이를 통해 소스코드 작성 단계에서 보일러코드를 제거하고 소스코드를 간결하게 작성할 수 있습니다. 컴파일 과정에서 어노테이션을 처리하여 데이터를 검증할 수 있으며, 소스코드를 추가하거나 변경할 수도 있기 때문입니다. 어노테이션은 크게 세 가지 용도로 사용됩니다.

  1. 컴파일 과정에서 에러나 경고를 감지하도록 컴파일러에게 정보를 제공
  2. 컴파일 과정에서 특정 코드나 파일을 생성/변경하도록 지시하도록 컴파일러에게 정보를 제공
  3. 런타임에서 특정 작업을 실행하도록 정보를 제공

타입 소거

타입 소거(Type Erasure)는 제네릭의 타입 파라미터를 구체화된 자료형으로 대체하는 작업을 의미합니다. 자바 컴파일러는 소스 코드를 컴파일할 때 타입 소거를 수행합니다.

타입 파라미터가 Unbounded Type인지 Bounded Type인지에 따라 소거 방식이 다릅니다.

(1) Unbounded Type의 타입소거

Unbounded Type은 형식 타입 파라미터로 전달받게될 실 타입 파라미터의 범위를 제한하지 않은 것을 의미합니다. <?>, <T> 형태의 타입 파라미터라고 생각하시면 됩니다.

컴파일러는 타입 파라미터를 소거하고 Unbounded Type인 <?>, <T>를 Object로 대체합니다.

(2) Bounded Type의 타입소거

Bounded Type은 형식 타입 파라미터로 전달되는 실 타입 파라미터의 범위를 제한한 것을 의미합니다. <E extends Something> 형태의 타입 파라미터라고 생각하시면 됩니다.
