[안드로이드] compileSdkVersion, minSdkVersion, targetSdkVersion 차이점

2019/01/16 카테고리 Android

Android의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 런타임에서 SDK 버전을 확인하는 방법에 대해 알아봤습니다. 이번 포스트에서는 compileSdkVersion, minSdkVersion, targetSdkVersion의 차이점에 대해 알아보겠습니다.

(1) compileSdkVersion

안드로이드 스튜디오에서 앱을 빌드할 때 어떤 SDK 버전을 사용할 것인지 지정합니다. 즉 comileSdkVersion은 지원할 수 있는 가장 높은 버전을 의미합니다.

안드로이드 스튜디오의 SDK Manager에 해당 SDK가 설치되어있어야 합니다.

(2) minSdkVersion

최소 지원 SDK 버전을 의미합니다. 즉 해당 버전 이하의 안드로이드 운영체제가 설치된 기기에서는 앱을 설치할 수 없습니다.

(3) targetSdkVersion

앱을 실제 기기에 설치하고 실행할 때 어떤 SDK 버전을 사용할지 지정합니다. 기능은 똑같아도 SDK 버전에 따라 다르게 동작하는 API도 있습니다. 이때는 targetSdkVersion 값에 맞는 API를 사용하게 됩니다.

targetSdkVersion을 지정하지 않으면 기본 값은 minSdkVersion과 동일하게 적용됩니다.

예제

모듈 수준의 build.gradle파일에서 다음과 같이 SDK 버전 설정을 설정합니다.

이는 SDK 버전이 25 이상 30 이하에서만 이 앱이 실행될 수 있음을 의미합니다. 또한 실제 동작은 27 버전으로 동작하게 됩니다. 보통 실무에서는

minSdkVersion <= targetSdkVersion == compileSdkVersion

로 SDK 버전을 설정합니다.

마치며

이번 포스트에서는 compileSdkVersion, minSdkVersion, targetSdkVersion 차이점에 대해 알아봤습니다. 다음 포스트에서는 ADB(Android Debug Bridge)에 대해 알아보겠습니다.

[안드로이드] 안드로이드 SDK란?

2019/01/15 카테고리 Android

Android의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 구글 플레이 스토어에 앱을 출시하는 방법에 대해 알아봤습니다. 이번 포스트에서는 안드로이드 SDK에 대해 알아보겠습니다.

안드로이드 SDK

안드로이드 SDK(Software Development Kit)는 안드로이드 앱을 개발하기 위한 툴 킷입니다.

안드로이드 스튜디오에서는 SDK Manager를 제공하는데 이 곳에서 안드로이드 SDK의 설치나 삭제, 설정을 할 수 있습니다. 또한 SDK Manager에서 SDK의 설치 경로를 확인할 수 있습니다.

그럼 터미널에서 SDK의 설치 경로로 이동해보겠습니다.

SDK에 무엇이 설치되어있는지 확인해봅시다.

(1) system-images

system-images폴더에는 에뮬레이터에서 사용되는 안드로이드 운영체제 이미지가 있습니다.

(2) platforms

platforms폴더에는 안드로이드 플랫폼이 설치되어 있습니다.

(3) tools

tools폴더와 tools/bin폴더에는 앱을 개발하고 디버깅하기 위한 여러가지 툴이 있습니다. 에뮬레이터, SD 매니저, AVD 매니저, UI 뷰어 등이 이 폴더에 설치되어있습니다.

안드로이드 스튜디오에서도 tools폴더에 설치된 툴을 확인할 수 있습니다.

(4) build-tools

build-tools폴더에는 앱을 빌드하기 위한 도구들이 설치되어있습니다.

(5) platform-tools

platform-tools폴더에는 안드로이드 기기와 연결했을 때 사용하는 도구들이 설치되어있습니다. adb, fastboot, systrace, sqlite3 등이 이 폴더에 설치되어있습니다.

마치며

이번 포스트에서는 안드로이드 SDK에 대해 알아봤습니다. 다음 포스트에서는 런타임에서 SDK 버전을 확인하는 방법에 대해 알아보겠습니다.

앱 게시 취소

앱 게시가 취소되면 새로운 사용자는 플레이스토어에서 앱을 검색하거나 다운로드할 수 없습니다. 그러나 기존 사용자는 앱을 계속 사용할 수 있습니다.

우선 구글 플레이 콘솔(Google Play Console)에 접속합니다.

왼쪽 사이드바 메뉴 > All apps로 이동합니다. 이 곳에서 출시한 앱 목록을 확인할 수 있습니다. 삭제할 앱을 선택합니다.

Setup > Advanced settings로 이동합니다.

App availability를 Unpublished로 변경하고 Save changes버튼을 누르면 앱 게시가 취소됩니다.

변경 사항이 반영되는데 몇 시간이 걸릴 수 있습니다.

게시가 취소된 앱은 삭제되기 전까지 다시 출시할 수 있습니다.

앱 삭제

구글 플레이 콘솔 고객센터(Google Play Console Help)에 접속합니다.

지원이 필요한 사항을 입력합니다.

앱 삭제를 선택합니다.

이메일을 선택합니다.

충족 사항을 확인합니다.

이름, 성, 이메일 주소, 내 위치, 개발자 이름을 입력합니다.

개발자 계정 ID를 입력합니다. 계정 세부정보 페이지를 선택하면 개발자 계정 ID를 확인할 수 있습니다.


패키지 이름을 입력합니다.

패키지 이름은 구글 플레이 콘솔 > All apps에서 확인할 수 있습니다.

문제 해결에 도움이 되는 스크린샷을 추가합니다. 삭제할 앱의 스크린샷을 첨부합니다.

제출하면 처리되는데 1~2일 정도가 소요됩니다.

개발자 계정 만들기

구글 플레이스토어에 앱을 출시하려면 구글 개발자 계정을 만들어야 합니다. http://play.google.com/apps/publish/signup을 방문하여 구글 개발자 계정을 생성합니다. (현재는 미화로 25달러를 결제해야하며, 한번 결제 시 평생 이용할 수 있습니다.)

앱 생성하기

구글 플레이 개발자 콘솔 계정을 만들었다면 구글 플레이 콘솔(Google Play Console)에 접속합니다. 그리고 Create app 버튼을 클릭합니다.

App details의 항목들을 입력합니다.

  1. App name: 플레이 스토어에 출시되는 앱 이름
  2. Default language: 기본 언어
  3. App or game: 게임 앱인 경우 game, 그 외에는 App
  4. Free or paid: 무료 앱인 경우 free, 유료 앱인 경우 paid (무료 선택 시 추후에 유료로 변경할 수 없습니다.)

Declarations 항목들을 입력합니다.

개발자 프로그램 정책과 미국 수출법규에 동의한다는 내용입니다.

이제 오른쪽 하단의 create app 버튼을 클릭하면 앱이 생성됩니다. 생성된 앱은 왼쪽 사이드 바 메뉴에의 All apps를 선택하면 확인할 수 있습니다.

앱 설정하기

All apps에서 생성한 앱을 선택합니다.

왼쪽 사이드바 메뉴에 Dashboard를 클릭합니다.

Dashboard 페이지에서는 앱 출시를 위해 입력해야할 정보가 단계별로 기재되어 있습니다. 하나씩 선택하고 입력하면 됩니다.

또한 Dashboard에서는 각 단계의 진행상황을 확인할 수 있습니다.

1. Start testing now

2. Set up your app

다음 7개 항목을 입력해야합니다.

(a) App access


All functionality is available without special access를 체크하고 저장합니다. 특별한 접근 권한 없이 모든 이용이 가능하다는 뜻입니다.

(b) Ads


광고 여부를 체크합니다. 앱 내에 광고가 있으면 “Yes, my app contains adds”를, 광고가 없으면 “No, my app does not contain ads”를 선택하고 저장합니다.

(c) Content ratings

콘텐츠의 등급을 설정합니다. Start new questionaire를 클릭하여 설문을 시작합니다.

이 단계에서는 이메일 주소와 앱 카테고리, 앱의 적합성을 조사합니다.

모든 단계가 끝나면 각 국가에서의 등급을 확인할 수 있습니다.

(d) Target audience and content

앱을 사용할 연령층과 어린이 관심 유도 여부 등을 확인합니다. 설문을 모두 완료합시다.

(e) News Apps

뉴스 앱인지를 체크합니다.

(f) App category & Contact

앱의 카테고리을 입력합니다.

개발자 연락처를 입력하고, 외부 마케팅 여부를 클릭합니다.

(g) Main store listing

앱 스토어에 표시될 앱 정보를 입력합니다.

  • App name: 플레이 스토어에 표시되는 앱 이름
  • Short description: 간단한 앱 설명
  • Full description: 자세한 앱 설명

  • App icon: 플레이스토어에 표시되는 앱 아이콘 이미지 (512x512px)
  • Feature graphic: 출시된 앱을 공유했을 때 나타나는 섬네일 이미지 (1024x500px)
