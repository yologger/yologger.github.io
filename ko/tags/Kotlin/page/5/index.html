객체지향 프로그래밍

Kotlin은 모든 대상을 객체(Object)로 바라봅니다. 이러한 점에서 Kotlin은 객체지향 프로그래밍 언어입니다. 객체지향 프로그래밍을 이해하려면 클래스와 인스턴스에 대해 알아야합니다.

클래스와 인스턴스

클래스와 인스턴스를 설명할 때 와플 기계과 와플을 예로 많이 듭니다.

클래스는 인스턴스를 만드는 틀, 인스턴스는 클래스로 만든 무언가라고 생각하면 됩니다. 즉 와플 기계는 클래스, 와플은 인스턴스입니다. 이 클래스를 코드로 표현하면 다음과 같습니다.

[Kotlin] 집합자료형(Collection) - List

2019/01/04 카테고리 Kotlin

Kotlin의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

집합자료형 (Collection)

집합자료형(Collection)을 사용하면 같은 자료형의 데이터를 여러 개 담을 수 있습니다. Kotlin은 List, Set, Map이라는 3개의 집합자료형을 지원합니다. List, Set, Map은 모두 Collection클래스를 상속하고 있습니다.

List

변경 불가능한 List를 선언할 때는 listOf()메소드를 사용합니다.

변경 가능한 List를 선언할 때는 mutableListOf()메소드를 사용합니다.

List는 Array와 매우 유사합니다. 하지만 차이점도 존재합니다. List는 초기화 후 값을 추가, 삭제할 수 있습니다. 반면 Array는 초기화하여 크기가 결정되면 값을 추가, 삭제할 수 없습니다.

집합자료형 조작 함수

집합자료형은 데이터 조작을 위한 다양한 메소드를 제공합니다. List의 대표적인 메소드 몇 가지만 알아보겠습니다.

1. filter()

2. sortedBy()

2. map()과 forEach()

[Kotlin] 배열(Array)

2019/01/03 카테고리 Kotlin

Kotlin의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 Kotlin의 자료형(Data Type)에 대해 알아봤습니다. 이번 포스트에서는 Kotlin의 배열(Array)에 대해 알아보겠습니다.

배열

배열은 같은 자료형의 데이터를 한꺼번에 관리하는데 사용합니다.

배열은 인스턴스가 생성될 때 크기가 설정되므로 동적으로 데이터를 추가하거나 삭제할 수 없습니다. 데이터를 동적으로 추가하거나 삭제할 땐 List를 사용합니다.

배열을 선언하고 초기화할 때는 arrayOf()메소드를 사용합니다. 담을 데이터의 자료형은 <>안에 적어줍니다.

배열을 초기화할 때 데이터를 직접 넣어줄 수 있습니다.

이렇게 선언된 배열의 각 요소에는 []를 사용하여 접근할 수 있습니다.

배열에는 유용하게 사용할 수 있는 속성과 메소드가 내장되어 있습니다.

키워드in은 요소가 배열에 포함되는지 확인하는데 사용할 수 있습니다.

마치며

이번 포스트에서는 배열(Array)에 대해 알아봤습니다. 다음 포스트에서는 집합자료형(Collection)과 List에 대해 알아보겠습니다.
