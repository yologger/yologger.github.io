[자바] 에러 처리(Exception & Error Handling)

2018/01/21 카테고리 Java

자바(Java)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 열거형(enum)에 대해 알아봤습니다. 이번 포스트에서는 에러 처리(Error Handling)에 대해 알아보겠습니다.

Throwable

자바는 오류가 발생하면 Error나 Exception을 발생시킵니다. Error와 Exception은 모두 Throwable클래스를 상속받습니다.

(1) Error

Error는 코드에서 잡아낼 수 없으며 어플리케이션 자체가 다운됩니다. 대표적으로 메모리가 부족하여 발생하는 OutOfMemoryError, StackOveflowError가 있습니다.

(2) Exception

Exception은 try-catch구문으로 잡아낼 수 있으며 프로그램이 다운되지 않도록 처리할 수 있습니다. 대표적으로 값이 null인 변수에 접근하면 발생하는 NullPointException이 있습니다.

이때 try-catch구문으로 Exception이 발생할 가능성이 있는 부분을 감싸주면 Exception이 발생해도 어플리케이션을 다운시키지 않고 에러 처리를 할 수 있습니다.

개발자가 직접 Exception을 정의하고 발생시킬 수 있습니다.

직접 Exception을 발생시킬 때는 키워드throw를 사용합니다.

Exception을 발생시킬 가능성이 있는 함수 뒤에는 키워드throws를 사용하여 가능성을 표기할 수 있습니다.

이 함수는 try-catch구문 안에서 사용해야합니다.

[자바] 인터페이스(Interface)

2018/01/17 카테고리 Java

자바(Java)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 추상 클래스(Abstract Class)에 대해 알아봤습니다. 이번 포스트에서는 인터페이스(Interface)에 대해 알아보겠습니다.

인터페이스

모든 메소드가 선언만 있고 구현부가 없는 클래스를 인터페이스라고 합니다. 인터페이스는 특정 규격을 준수하도록 강제하는데 사용합니다.

인터페이스를 선언할 때는 키워드 interface를 사용합니다. 인터페이스에는 메서드를 선언만 하고 구현은 하지 않습니다.

인터페이스는 인스턴스 생성할 수 없습니다.

따라서 인터페이스를 구현해야합니다. 인터페이스를 구현한 클래스를 구현체(Imlementation)라고 합니다. 인터페이스를 구현할 때는 키워드 implements를 사용합니다.

인터페이스 Database 규격을 따르는 클래스 두 개를 구현해보겠습니다.

위 예제에서는 Database라는 규격을 따르도록 강제하고 있습니다. 이렇게 하면 Database를 사용하는 입장에서 실제 구현체가 무엇인지 신경쓰지 않고 사용할 수 있습니다. 이를 어려운 용어로 다형성(Polymorphism)이라고 합니다.

다형성을 예제를 통해 살펴보겠습니다.

변수 db에 MySQL의 인스턴스가 할당되든 Realm의 인스턴스가 할당되는 상관없이 save()메소드를 호출할 수 있습니다. MySql클래스와 Realm클래스가 Database인터페이스의 표준을 따르고 있기 때문입니다.

이처럼 인터페이스와 다형성을 사용하면 유지보수성을 향상시킬 수 있습니다.

마치며

이번 포스트에서는 인터페이스(Interface)에 대해 알아봤습니다. 다음 포스트에서는 내부 클래스(Inner Class)에 대해 알아보겠습니다.

[자바] 키워드 final

2018/01/15 카테고리 Java

자바(Java)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 상속(Inheritance)에 대해 알아봤습니다. 이번 포스트에서는 키워드 final에 대해 알아보겠습니다.

final

키워드final은 크게 세 가지 용도로 사용됩니다.

(1) 상수

키워드 final은 변수를 변경할 수 없는 상수로 만드는데 사용합니다.

이렇게 선언된 상수는 값을 변경할 수 없습니다.

(2) 상속 불가능한 클래스

부모 클래스에 final을 붙여보겠습니다.

자식 클래스는 상속이 불가능합니다.

(3) 오버라이드 불가능한 메소드

부모 클래스 메소드 앞에 final을 붙으면 오버라이딩이 불가능합니다.

마치며

이번 포스트에서는 키워드final에 대해 알아봤습니다. 다음 포스트에서는 추상 클래스(Abstract Class)에 대해 알아보겠습니다.

[자바] 정적 멤버변수(Static Variable)과 정적 메소드(Static Method)

2018/01/12 카테고리 Java

자바(Java)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 클래스의 생성자, 멤버변수, 메소드에 대해 알아봤습니다. 이번 포스트에서는 클래스의 정적 멤버변수(Static Variable)와 정적 메소드(Static Method)에 대해 알아보겠습니다.

static

키워드 static은 클래스의 멤버변수와 메소드 앞에 붙일 수 있습니다. 이렇게 static이 붙은 멤버변수와 메소드를 정적 멤버변수와 정적 메소드라고 합니다.

정적 멤버변수와 정적 메소드는 다음과 같은 특징이 있습니다.

  • 클래스의 인스턴스를 생성하지 않고도 정적 메소드를 호출할 수 있습니다.
  • 클래스의 인스턴스를 생성하지 않고도 정적 멤버변수에 접근할 수 있습니다.
  • 클래스의 모든 인스턴스가 정적 멤버변수를 공유합니다.

정적 멤버변수와 정적 메소드는 키워드 static을 붙여서 선언합니다.

정적 멤버변수와 정적 메소드는 다음과 같이 인스턴스를 생성하지 않고 사용할 수 있습니다.
