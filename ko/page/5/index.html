2018/01/20 카테고리 Kotlin

Nullable타입의 변수에 접근하려면 값이 null이 아닌지를 체크해야합니다. 그렇지 않으면 컴파일 자체가 안됩니다.

Null 체크

우선 변수가 Nullable타입이라고 단언하고 변수에 접근할 수 있습니다. 다만 실제 값이 null인 경우 런타임 에러가 발생합니다.

따라서 런타임 에러를 방지하기 위해 if문으로 null체크를 합니다.

Safe Call (?.)

매번 if문으로 null체크를 하는 것은 번거로운 일입니다. 따라서 보통 단축구문인 물음표(?)를 사용하며, 이를 Safe Call이라고 합니다.

[Kotlin] null 처리 - (1)

2018/01/19 카테고리 Kotlin

들어가며

이번 포스트에서는 코틀린의 null 처리에 대해 알아보겠습니다.

Not null 타입

코틀린은 변수를 초기화하지 않으면 컴파일 자체가 안됩니다.

클래스의 멤버변수 역시 초기화하지 않으면 컴파일 자체가 안됩니다.

Not null 타입의 변수에는 null을 할당할 수도 없습니다.

Nullable 타입

null을 할당하기 위해서는 변수를 Nullable타입으로 선언해야합니다. 이때는 타입 뒤에 물음표(?)를 붙입니다.

물론 Nullable타입이라도 초기화하지 않으면 컴파일 자체가 되지 않습니다.
