[Kotlin] 배열(Array)

2019/01/03 카테고리 Kotlin

Kotlin의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 Kotlin의 자료형(Data Type)에 대해 알아봤습니다. 이번 포스트에서는 Kotlin의 배열(Array)에 대해 알아보겠습니다.

배열

배열은 같은 자료형의 데이터를 한꺼번에 관리하는데 사용합니다.

배열은 인스턴스가 생성될 때 크기가 설정되므로 동적으로 데이터를 추가하거나 삭제할 수 없습니다. 데이터를 동적으로 추가하거나 삭제할 땐 List를 사용합니다.

배열을 선언하고 초기화할 때는 arrayOf()메소드를 사용합니다. 담을 데이터의 자료형은 <>안에 적어줍니다.

배열을 초기화할 때 데이터를 직접 넣어줄 수 있습니다.

이렇게 선언된 배열의 각 요소에는 []를 사용하여 접근할 수 있습니다.

배열에는 유용하게 사용할 수 있는 속성과 메소드가 내장되어 있습니다.

키워드in은 요소가 배열에 포함되는지 확인하는데 사용할 수 있습니다.

[Java] Java Platform(JDK) 버전과 종류

2018/04/02 카테고리 Java

Java의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

Java Platform(JDK)에는 여러 에디션이 존재합니다. 이번 포스트에서는 대표적인 3가지 에디션의 특징에 대해 알아보겠습니다.

Java SE (Java Standard Edition)

  1. 과거에는 J2SE(Java 2 SE)로 불렸다.
  2. 가장 보편적으로 사용되는 에디션입니다.
  3. 자바 데스크탑, 모바일 어플리케이션 개발에 사용됩니다.
  4. JDK 외에 Swing, AWT 등 데스크탑 개발에 필요한 API가 포함됩니다.

Java EE (Java Enterprise Edition)

  1. 과거에는 J2EE(Java 2 EE)로 불렸습니다.
  2. 자바 서버 애플리케이션 에 사용된다.
  3. Java SE의 모든 API도 포함된다.
  4. 그 외 EJB, JSP, Sevlet, JDBC등 서버 어플리케이션 개발을 위한 API가 포함된다.

Java ME (Java Micro Edition)

  1. Java SE API의 일부만 포함
  2. 셋톱박스, PDA등 임베디드 시스템을 개발하는데 사용됩니다.

[Java] 람다식(익명함수)와 함수형 인터페이스

2018/03/16 카테고리 Java

Java의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 1급 객체와 1급 함수, 그리고 함수형 프로그래밍에 대해 알아봤습니다. 이번 포스트에서는 람다식(익명함수)와 함수형 인터페이스에 대해 알아보겠습니다.

람다식(Lambda Expression)

람다식(Lambda Expression)이란 함수를 하나의 식(expression)으로 표현한 것입니다. 함수를 람다식으로 표현하면 메소드의 이름이 필요 없기 때문에, 람다식은 익명 함수(Anonymous Function)라고도 불립니다.

코틀린은 함수를 일급객체로 취급하므로 람다식을 다음 코드와 같이 변수에 저장할 수 있습니다. (일급함수에 대해서는 이 포스트에서 확인할 수 있습니다.) 이는 언어 자체에서 함수 타입을 지원하기 때문입니다. 정수 타입에는 정수를 할당하는 것처럼 함수 타입에는 함수를 할당할 수 있습니다.

자바에서의 람다식

자바(Java)는 코틀린(Kotlin), 스위프트(Swift)같은 언어에 비해 비교적 오래되었습니다. 또한 객체지향 프로그래밍을 패러다임으로 만들어졌기에 람다식과 함수형 프로그래밍을 지원하지 않았습니다. 그러나 2014년 발표된 Java 8부터 함수형 프로그래밍을 지원하기 위한 Java API가 추가되었습니다.

아쉽게도 자바(Java)는 함수 타입이 없으므로 람다식을 변수에 저장하거나 매개변수로 전달할 수 없습니다. 이를 해결하기 위해 도입된 개념이 @FunctionalInterface어노테이션 입니다.

@FunctionInterface

함수형 인터페이스를 구현하기 위해서는 인터페이스를 정의하고 그 내부에는 1개 뿐인 abstract 함수를 선언하고, @FunctionalInterface 어노테이션을 붙여주면 됩니다.

이제 마치 변수에 함수를 할당하는 것처럼 사용할 수 있습니다.

Java API에서는 자주 사용될 것 같은 함수형 인터페이스가 이미 정의되어있습니다.

1. Supplier

Supplier는 매개변수 없이 반환값만을 갖는 함수형 인터페이스입니다. Supplier는 T get()을 추상 메소드로 갖고 있습니다.

2. Consumer

객체 T를 매개변수로 받아서 사용하며, 반환값은 없는 함수형 인터페이스입니다. Consumer는 void accept(T t)를 추상메소드로 갖고 있습니다.

3. Function <T, R>

Function은 객체 T를 매개변수로 받아서 처리한 후 R로 반환하는 함수형 인터페이스입니다. Function은 R apply(T t)를 추상메소드로 가지고 있습니다.

4. Predicate

Predicate는 객체 T를 매개 변수로 받아 처리한 후 Boolean을 반환합니다. Supplier는 Boolean test(T t)을 추상 메소드로 갖고 있습니다.

안드로이드에서의 람다식

람다식은 안드로이드 어플리케이션 개발에서도 자주 사용합니다. 보통 위젯을 클릭했을 때 실행할 구문을 람다식 안에 구현합니다.

우선 람다식을 사용하지 않고 익명 클래스를 사용하여 특정 버튼을 클릭했을 때 실행할 구문을 등록해보겠습니다.
