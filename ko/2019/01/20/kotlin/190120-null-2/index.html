4개의 포스트를 통해 Kotlin의 null 처리 방법에 대해 알아보고 있습니다. 이번 두 번째 포스트에서는 Null Check 방법에 대해 알아보겠습니다.

Nullable

Nullable타입의 변수에 접근하려면 값이 null이 아닌지를 체크해야합니다. 그렇지 않으면 컴파일 자체가 안됩니다.

Null Check

우선 변수가 Nullable타입이라고 단언하고 변수에 접근할 수 있습니다.

다만 값이 null인 경우 런타임 에러가 발생합니다.

따라서 런타임 에러를 방지하기 위해 if문으로 null체크를 합니다.

Safe Call (?.)

매번 if문으로 null체크를 하는 것은 번거로운 일입니다. 따라서 보통 단축구문인 물음표(?)를 사용하며, 이를 Safe Call이라고 합니다.
