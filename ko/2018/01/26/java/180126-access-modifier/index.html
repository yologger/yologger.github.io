[Java] 접근 제한자(public, private, protected, default)

2018/01/26 카테고리 Java

Java의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 Java의 형 변환(Type Casting)에 대해 알아봤습니다. 이번 포스트에서는 접근 제한자(Access Modifier)에 대해 알아보겠습니다.

접근 제한자 (Access Modifiers)

접근 제한자는 외부에서 클래스 내부의 메소드나 멤버변수에 접근하는 것을 제한하는 것입니다. Java는 private, protected, default, public이라는 4개의 접근 제한자를 지원합니다. 접근 제한자는 클래스, 생성자, 메소드, 멤버변수 앞에 붙일 수 있습니다.

private

private은 같은 클래스 내에서만 접근이 가능합니다. 클래스에는 붙일 수 없으며 멤버변수, 메소드, 생성자 앞에만 붙일 수 있습니다.

default

접근 제한자를 생략하면 기본적으로 default 접근 제한자가 적용됩니다. default는 동일한 package 내에서만 접근을 허용합니다. 멤버변수, 메소드, 생성자 뿐만 아니라 클래스 앞에도 붙일 수 있습니다.

Person클래스는 com.yologger.person에 정의되어있고 접근 제한자는 default입니다.

따라서 다른 패키지인 com.yologger.app에서 접근할 수 없습니다.

protected

protected는 같은 패키지에서 상속 관계에 있는 클래스에서만 접근을 허용합니다. 클래스에는 붙일 수 없으며 멤버변수, 메소드, 생성자 앞에만 붙일 수 있습니다.

public

public은 다른 패키지라도 어디에서든 접근이 가능합니다.

다른 패키지인 com.yologger.app에서도 접근할 수 있습니다.
