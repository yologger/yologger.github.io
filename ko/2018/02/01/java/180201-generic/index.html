제네릭

제네릭(Generic)은 클래스나 메서드 내부에서 사용할 데이터의 자료형을 외부에서 설정하는 것입니다. 제네릭을 사용하면 다음과 같은 장점이 있습니다.
  1. 특정 타입에 종속되지 않은 유연한 로직
  2. 타입 안정성
  3. 자동 형변환

제너릭을 사용하지 않았을 때의 문제점

제네릭을 사용하면 특정 타입에 종속되지 않은 유연한 로직을 구현할 수 있습니다.

아래 Stack클래스 예제를 살펴봅시다.

Stack클래스는 String타입의 배열을 멤버변수로 가지고 있습니다. 따라서 다음과 같이 String타입의 데이터만 추가하고 삭제할 수 있습니다.

이제 Integer타입의 데이터를 추가하려고 합니다. 그러나 위 Stack클래스를 사용할 수는 없습니다. Stack클래스는 String타입의 데이터만 입력받고 저장하고 삭제할 수 있기 때문입니다. 따라서 별도의 StackForInteger클래스를 정의해야합니다.

이제 StackForInteger클래스를 사용해서 Integer타입의 데이터를 추가하고 삭제할 수 있게 되었습니다.

이제 String, Integer뿐만 아니라 더 다양한 타입의 데이터를 취급하려고 합니다. 그럼 타입이 추가될 때 마다 새로운 클래스를 정의해야 할까요? 바로 이러한 경우 제네릭을 사용할 수 있습니다.

(장점 1) 특정 타입에 종속되지 않은 유연한 로직

아래 코드는 제너릭을 적용한 Stack클래스 입니다.

클래스 이름 옆 <>안의 문자E를 타입 파라미터(Type Parameter)라고 합니다. 타입 파라미터는 관습적으로 대문자 E(Element), T(Type), K(Key), V(Value)를 사용합니다.

그리고 타입 파라미터가 붙어있는 클래스를 제너릭 클래스라고 합니다. 제너릭 클래스는 인스턴스를 생성할 때 자료형 정보를 타입 파라미터에 전달합니다.

우선 타입 파라미터로 String을 전달하여 Stack을 생성해봅시다.

이제 String타입의 데이터를 추가할 수 있습니다.

타입 파라미터로 Integer를 전달하여 새로운 Stack을 생성해봅시다.

이제 Integer타입의 데이터를 추가할 수 있습니다.

이처럼 제너릭을 사용하면 특정 타입에 종속되지 않은 유연한 로직을 작성할 수 있습니다.

(장점 2) 타입 안정성

타입 안정성은 런타임에 발생할 에러를 컴파일 타임에 찾아낼 수 있음을 의미합니다.

아래 예제를 살펴봅시다. 자바 API에서 기본적으로 제공하는 ArrayList를 쉽고 간단하게 재구현했습니다.

이제 CustomArrayList의 인스턴스를 생성하고 정수형 데이터를 넣어보겠습니다.

컴파일도 잘 되고 런타임에서도 잘 동작하는 것을 확인할 수 있습니다. add()메소드는 매개변수로 Object타입의 데이터를 입력받기 때문입니다.

이제 add()메소드에 문자열 데이터를 전달해봅시다.

add()메소드는 매개변수로 Object타입의 데이터를 입력받기 때문에 문법적으로는 아무런 문제가 없고 컴파일도 정상적으로 수행됩니다. 그러나 실행을 하면 런타임에서 다음과 같은 오류가 발생합니다.

형 변환이 제대로 이루어지지 않았다는 오류 메시지입니다.

오류는 컴파일 타임에서 찾아내는 것이 가장 좋습니다. 런타임에 오류가 발생하여 어플리케이션이 다운되면 비즈니스에 큰 문제가 생기기 때문입니다.

이러한 문제점을 해결하는데 제너릭을 사용할 수 있습니다. 제네릭을 사용하여 CustomArrayList를 정의해보겠습니다.

이제 인스턴스를 생성할 때 자료형을 전달하면, 컴파일러는 타입 파라미터로 전달된 자료형을 클래스 내부의 타입 파라미터 T로 대체되는 것처럼 인식하게 됩니다.

또한 잘못된 타입의 데이터를 입력하면 컴파일 시점에 오류를 탐지하여 타입 안정성을 높여줄 수 있습니다.

(장점 3) 자동 형변환

제네릭을 사용하면 자동으로 형변환을 해준다는 장점이 있습니다. 제네릭을 사용하지 않은 CustomArrayList클래스를 다시 살펴봅시다.

메소드 get()은 Object타입의 데이터를 반환합니다. 따라서 값을 사용하려면 형 변환을 해야합니다.

이제 제네릭을 사용하여 CustomArrayList클래스를 정의합시다.

제네릭을 사용하면 별도의 형 변환이 필요없습니다.
