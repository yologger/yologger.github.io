[코틀린] 접근 제한자(Access Modifiers)

2019/02/06 카테고리 Kotlin

코틀린(Kotlin)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

접근 제한자

접근 제한자(Access Modifiers)는 외부에서 클래스 내부의 메소드나 멤버변수에 접근하는 것을 제한하는 것입니다. 코틀린은 네 개의 접근 제한자를 지원합니다.

(1) private

private으로 선언된 변수는 해당 클래스 내부에서만 접근이 가능합니다.

private으로 선언된 변수는 상속받는 자식 클래스에서도 접근이 불가능합니다.

(2) protected

protected로 선언된 변수는 해당 클래스와 자식 클래스에서 접근이 가능합니다.

(3) internal

internal로 선언된 변수는 같은 모듈 내에서는 어디서 접근이 가능합니다. 코틀린 공식 문서에서 말하는 같은 모듈은 아래 상황을 뜻합니다.

  1. Android Studio Module
  2. IntelliJ IDEA Module
  3. Maven Project

쉽게 말하면 안드로이드 스튜디오 프로젝트의 같은 모듈에서는 접근이 가능하다고 보면 되겠습니다.

(4) public (default)

어디서든 접근이 가능합니다. 접근 제한자를 붙이지 않으면 기본적으로 public입니다.

마치며

이번 포스트에서는 접근 제한자에 대해 알아봤습니다. 다음 포스트에서는 리플렉션(Reflection)에 대해 알아보겠습니다.

TedPermission 라이브러리

TedPermission는 대한민국의 개발자 박상권님께서 개발하신 권한 설정 라이브러리입니다. 이 라이브러리를 사용하면 권한 설정을 좀 더 쉽게 구현할 수 있습니다.

설치

모듈 수준의 build.gradle파일에 다음과 같이 의존성을 추가합니다.

build.gradle(Module: my_project.app)

사용 방법

우리는 이전 포스트에서 카메라의 권한을 확인하고, 권한이 없으면 요청하고, 권한을 얻었을 때 카메라를 여는 예제를 살펴봤습니다.

이제 동일한 내용을 TedPermission 라이브러리를 사용해서 구현해보겠습니다.

이제 앱을 실행하고 버튼을 누르면 다음과 같이 권한 설정 대화상자가 나옵니다.

앱 사용 중에만 허용을 누르면 onPermissionGranted()메소드가 호출되며, 앱을 종료하고 다시 시작해도 권한 설정 대화가 나오지 않습니다. 이번만 허용을 눌러도 onPermissionGranted() 메소드가 호출됩니다. 차이점은 앱 종료 후 다시 시작할 때마다 권한 설정 대화상자가 다시 나옵니다.

만약 거부를 누르면 setDeniedMessage()메소드에서 설정한 메시지와 함께 설정 또는 닫기를 선택할 수 있는 대화상자가 나옵니다.

여기서도 닫기를 누르면 onPermissionDenied()가 호출됩니다. 만약 설정을 누르면 다음과 같이 앱 설정 화면으로 이동합니다.

여기서 권한을 모두 부여하고 뒤로가기를 누르면 onPermissionGranted()이 호출됩니다. 그러나 하나라도 권한을 부여하지 않으면 onPermissionDenied()이 호출됩니다.

[코틀린] 일급 객체, 함수형 프로그래밍

2019/02/03 카테고리 Kotlin

코틀린(Kotlin)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 람다식과 익명함수에 대해 알아봤습니다. 이번 포스트에서는 일급 객체(First-class Object), 함수형 프로그래밍(Functional Programming)에 대해 알아보겠습니다.

일급 객체

다음 세 가지 조건을 모두 충족하면 일급 객체라고 힙니다.

  1. 변수나 상수에 할당할 수 있다.
  2. 함수의 인자로 전달할 수 있다.
  3. 함수의 반환값으로 반환할 수 있다.

코틀린에서는 함수를 일급 객체로 취급합니다. 함수가 위의 세 가지 조건을 모두 충족하는지 살펴보겠습니다.

(1) 함수를 변수나 상수에 할당할 수 있다.

코틀린에서는 함수를 변수에 할당할 수 있습니다. 이때 리플렉션(::)을 사용합니다.

람다식 또한 변수에 할당할 수 있습니다.

(2) 함수를 함수의 인자로 전달할 수 있다.

코틀린에서는 함수를 함수의 인자로 전달할 수 있습니다.

(3) 함수를 함수의 반환 값으로 반환

코틀린에서는 함수를 함수의 반환 값으로 반환할 수 있습니다.

함수 타입

함수 타입은 위 세 가지 특성을 가능하게 해주는 코틀린의 자료형입니다. 정수형 타입의 변수에 정수를 할당하는 것 처럼 함수형 타입에는 함수를 할당할 수 있습니다.

위 예제에서는 (Int, Int) -> Int가 바로 함수 타입입니다.

마치며

이번 포스트에서는 일급 객체, 함수형 프로그래밍에 대해 알아봤습니다. 다음 포스트에서는 Unit에 대해 알아보겠습니다.

[코틀린] 에러 처리(Exception & Error Handling)

2019/02/01 카테고리 Kotlin

코틀린(Kotlin)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 코틀린의 형 변환(Type Casting)에 대해 알아봤습니다. 이번 포스트에서는 코틀린의 에러 처리(Error Handling)에 대해 알아보겠습니다.

Throwable

코틀린은 오류가 발생하면 Error나 Exception을 발생시킵니다. Error와 Exception은 모두 Throwable클래스를 상속받습니다.

Error

Error는 코드에서 잡아낼 수 없으며 어플리케이션 자체가 다운됩니다. 대표적으로 OutOfMemoryError, StackOveflowError가 있습니다.

Exception

Exception은 try-catch구문으로 잡아낼 수 있으며, 프로그램이 다운되지 않도록 처리할 수 있습니다. 대표적으로 값이 null인 변수에 접근할 때 발생하는 NullPointException이 있습니다.

개발자가 직접 Exception을 정의하고 발생시킬 수도 있습니다.

직접 Exception을 발생시킬 때는 키워드 throw를 사용합니다.

Exception을 발생시킬 수 있는 메소드에는 @Throws 어노테이션을 붙여 에러 가능성을 표기할 수 있습니다.

@Throws어노테이션이 붙은 메소드는 try-catch문 내에서 호출하는 걸 권장합니다.
