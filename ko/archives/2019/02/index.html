[Kotlin] 제너릭 (Generic)

2019/02/14 카테고리 Kotlin

제너릭 (Generic)

제네릭 (Generic)은 클래스나 메서드 내부에서 사용할 데이터의 타입을 외부에서 설정하는 것입니다. 이를 통해 컴파일 시 메서드나 클래스에 사용되는 다양한 데이터의 타입 체크를 가능하게 합니다.

클래스에서의 제너릭 (Generic)

제네릭 클래스를 정의할 땐 <>안에 임의의 타입 파라미터를 명시합니다. 보통 타입 파라미터는 관습적으로 대문자 E(Element), T(Type), K(Key), V(Value)를 사용합니다.

이렇께 정의된 제너릭 클래스는 다음과 같이 사용합니다. <>안의 타입 파리미터 T에는 어떠한 데이터 타입도 들어갈 수 있습니다.

여러 개의 타입 파라미터을 사용할 수도 있습니다.

타입 제한

키워드 where를 사용할 수도 있다.

객체를 생성한다는 것은 메모리를 그만큼 사용한다는 것입니다. 만약 이 함수를 10번 호출하면 10개의 객체가 생성되며, 이러한 부분이 성능을 떨어뜨릴 수 있습니다.

위 코드를 컴파일하면 객체를 생성하지 않고 코드 자체를 호출하는 부분으로 복사합니다.

inline 함수를 사용하면 메모리 성능을 향상시킬 수 있습니다. 하지만 코드 양이 많은 함수를 inline하면 컴파일된 코드의 양도 많아질 수 있습니다. 따라서 inline 함수는 1~3줄 정도의 함수에 사용하는 것을 권장합니다.

주의할 점

하지만 인라인 함수는 내부적으로 코드를 복사하기 때문에, 인자로 전달받은 함수를 다른 함수의 인자로 전달하거나 다시 참조할 수 없습니다.

위 코드는 다음과 같은 컴파일 에러를 발생시킵니다.

noinline

그럼 위 예제에서 func2() 함수만 접근이 가능하고 func2() 함수만 다른 함수의 인자로 전달되도록 할 수는 없을까요? 바로 이러한 경우 키워드 noinline를 사용합니다. 키워드 noinline붙은 키워드는 컴파일 시 일반 함수와 동일하게 동작합니다. 따라서 다시 함수에 접근하거나 다른 함수의 인자로 전달할 수 있습니다.
