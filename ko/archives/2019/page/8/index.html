람다식(Lambda Expression)은 익명함수(Anonymous Function)라고도 불리며 **이름이 없는 함수**를 의미합니다.

람다식을 사용할 때는 {}를 사용합니다.

[코틀린] 에러 처리(Exception & Error Handling)

2019/02/01 카테고리 Kotlin

코틀린(Kotlin)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 코틀린의 형 변환(Type Casting)에 대해 알아봤습니다. 이번 포스트에서는 코틀린의 에러 처리(Error Handling)에 대해 알아보겠습니다.

Throwable

코틀린은 오류가 발생하면 Error나 Exception을 발생시킵니다. Error와 Exception은 모두 Throwable클래스를 상속받습니다.

Error

Error는 코드에서 잡아낼 수 없으며 어플리케이션 자체가 다운됩니다. 대표적으로 OutOfMemoryError, StackOveflowError가 있습니다.

Exception

Exception은 try-catch구문으로 잡아낼 수 있으며, 프로그램이 다운되지 않도록 처리할 수 있습니다. 대표적으로 값이 null인 변수에 접근할 때 발생하는 NullPointException이 있습니다.

개발자가 직접 Exception을 정의하고 발생시킬 수도 있습니다.

직접 Exception을 발생시킬 때는 키워드 throw를 사용합니다.

Exception을 발생시킬 수 있는 메소드에는 @Throws 어노테이션을 붙여 에러 가능성을 표기할 수 있습니다.

@Throws어노테이션이 붙은 메소드는 try-catch문 내에서 호출하는 걸 권장합니다.

마치며

이번 포스트에서 에러 처리(Exception & Error Handling에 대해 알아봤습니다. 다음 포스트에서는 람다식과 익명함수에 대해 알아보겠습니다.

코틀린(Kotlin)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

var

변수를 선언할 때 키워드 var를 사용합니다. 변수는 런타임에 값을 수시로 변경할 수 있습니다.

val

val은 런타임에 값이 결정되는 상수입니다. 상수는 한 번 값이 할당되면 변경할 수 없습니다.

val는 클래스의 인스턴스도 할당할 수 있습니다.

const val

const val는 컴파일 타임에 값이 결정되는 상수입니다. 인스턴스를 할당할 수 없으며, 기본 자료형과 문자열만 할당할 수 있습니다. 상수의 이름은 통상 대문자와 언더바를 통해 선언합니다.

클래스 안에서 사용할 때는 companion object 블럭 안에 정의합니다.

코틀린 표준 라이브러리는 객체의 컨텍스트 내에서 코드 블럭을 실행하기 위한 목적만을 가진 다섯 개의 스코프 함수(Scope function)를 제공합니다. run, let, apply, also, with가 그것입니다. 다섯 개의 포스트에서 스코프 함수에 대해 알아보고있습니다.

  1. run()
  2. let()
  3. apply()
  4. also()
  5. with() ⬅️ ⬅️ ⬅️

with()

with()는 run()과 유사합니다. 차이점은 인스턴스 뒤에 점(.)을 붙여 호출하지 않고 함수의 인자로 인스턴스를 전달합니다.

개발을 하다보면 다음과 같이 반복적으로 객체의 속성에 접근하거나 객체의 메소드를 호출하는 경우가 많습니다.

이때 with()를 유용하게 사용할 수 있습니다.

마치며

우리는 다섯 개의 포스트에서 스코프 함수(Scope Function)에 대해 알아봤습니다. 다음 포스트에서는 코틀린에서의 this에 대해 알아보겠습니다.

also()

also()는 함수를 호출하는 객체가 인자의 형태로 코드 블록으로 전달된다는 점에서 apply()함수와 유사합니다. 그리고 전달 받은 객체가 반환됩니다.

인자가 하나인 경우 it을 사용하여 코드를 더 간략하게 할 수 있습니다.
