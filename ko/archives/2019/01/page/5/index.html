build.gradle 파일

build.gradle은 그래들 설정 파일입니다. 이 파일에 어떤 식으로 앱을 빌드할 지를 명시합니다. 그래들은 이 파일을 참조하여 프로젝트를 빌드합니다.

안드로이드 프로젝트를 생성하면 기본적으로 두 개의 build.gradle가 생성됩니다.

1. 프로젝트 수준의 build.gradle

하나의 프로젝트는 여러 개의 모듈(Module)로 구성될 수 있습니다. 프로젝트 수준의 build.gradle은 프로젝트 전체에, 다시 말해 모든 모듈에 적용되는 그래들 설정 파일입니다.

2. 모듈 수준의 build.gradle

모듈 수준의 build.gradle은 특정 모듈에만 적용되는 설정 파일입니다.

plugins 섹션에는 안드로이드 플러그인과 코틀린 플러그인의 사용을 선언하고 있습니다.

android 섹션에서는 빌드할 때 사용할 SDK와 빌드 도구의 버전을 정의하고 있습니다.

defaultConfig 섹션에 선언된 항목들은 빌드하는 동안 해당 모듈의 AndroidManifest.xml파일로 생성되는 요소를 정의합니다.

buildType 섹션에는 어플리케이션을 어떻게 빌드할 지를 명시합니다.

dependencies 섹션은 외부 모듈이나 라이브러리를 사용할 때 정의합니다. 같은 프로젝트 내 다른 모듈, jar 파일, 다른 개발자가 만든 라이브러리 등을 사용하려면 이 섹션에 명시해야합니다.

CoC

그래들은 설정보다는 관례(CoC, Convention over Configuration)라는 개념을 구현하고 있습니다. 즉, 그래들은 사전에 정의된 합리적인 기본 설정값을 가지고 있습니다. 따라서 모든 설정을 설정하지 않아도 기본 설정값이 적용됩니다. 이 덕분에 개발자는 모든 설정을 모두 설정할 필요가 없어집니다. 자신의 요구 사항에 맞지 않는 설정만 빌드 설정 파일에서 변경하면 됩니다.

배열

배열은 같은 타입의 데이터를 한꺼번에 관리하는데 사용합니다.

배열은 인스턴스가 생성될 때 크기가 설정되므로 동적으로 데이터를 추가하거나 삭제할 수 없습니다. 데이터를 동적으로 추가하거나 삭제할 땐 List를 사용합니다.

배열을 선언하고 초기화할 때는 arrayOf()메소드를 사용합니다. 담을 데이터의 타입은 제너릭(<>)안에 적어줍니다.

배열을 초기화할 때 데이터를 직접 넣어줄 수 있습니다.

이렇게 선언된 배열의 각 요소에는 []를 사용하여 접근할 수 있습니다.

배열에는 유용하게 사용할 수 있는 속성과 메소드가 내장되어 있습니다.

키워드in은 요소가 배열에 포함되는지 확인하는데 사용할 수 있습니다.

[안드로이드] 그래들(Gradle) - (3) 그래들 래퍼(Gradle Wrapper)

2019/01/03 카테고리 Android, Gradle

안드로이드(Android)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

우리는 이전 포스트에서 그래들(Gradle)에 대해 알아봤습니다. 이번 포스트에서는 그래들 래퍼(Gradle Wrapper)에 대해 알아보겠습니다.

Gradle Wrapper

안드로이드 스튜디오를 사용하는 개발자는 의문이 생길 수도 있습니다. 그래들을 별도로 설치하지 않았지만 프로젝트를 빌드할 수 있기 때문입니다. 이를 가능하게 하는 것이 그래들 래퍼입니다.

그래들 래퍼(Gradle Wrapper)는 그래들을 설치하지 않아도 프로젝트를 빌드할 수 있도록 프로젝트 내부에 그래들 관련 파일을 포함시키는 방법입니다. 그래들 래퍼 덕분에 기존의 프로젝트를 새로운 환경에 설치할 때 그래들을 별도로 설치 않아도 됩니다.

우리는 이전 포스트에서 그래들을 설치하고 아래와 같은 명령어를 입력하여 프로젝트를 빌드했습니다.

하지만 그래들 명령어를 사용하려면 빌드하는 컴퓨터에 그래들이 설치되어있어야 합니다. 이제 그래들 래퍼를 사용하여 프로젝트를 빌드해보겠습니다.

Gradle Wrapper 초기화

프로젝트 루트 디렉토리에서 다음 명령어를 입력합니다.

디렉토리에 그래들 래퍼와 관련된 파일이 생성되었습니다.

Gradle Wrapper 구조

생성된 디렉토리의 구조는 다음과 같습니다.

(1) gradlew

유닉스용 그래들 명령어 실행 스크립트 파일입니다.

(2) gradlew.bat

윈도우용 그래들 명령어 실행 스크립트 파일입니다.

(3) gradle/wrapper/gradle-wrapper.jar

그래들 래퍼 파일입니다. 위 두 개의 스크립트 파일 중 하나가 실행되면 운영체제에 맞는 환경을 로컬 컴퓨터에 다운받은 뒤 실제 태스크를 실행합니다.

(4) gradle/wrapper/gradle-wrapper.properties

그래들 래퍼 설정 파일입니다.

Gradle Wrapper를 통한 빌드

유닉스 계열인 Mac OS 환경에서 프로젝트를 빌드해보겠습니다. 다음 명령어를 실행하면 $ gradle build 명령어와 동일하게 태스크를 실행합니다.

안드로이드 스튜디오 프로젝트와 Gradle Wrapper

안드로이드 스튜디오에서 프로젝트를 생성하면 자동으로 Gradle Wrapper 관련 파일이 생성됩니다.

마치며

이번 포스트에서는 그래들 래퍼(Gradle Wrapper)에 대해 알아봤습니다. 다음 포스트에서는 build.gradle 파일에 대해 알아보겠습니다.

코틀린(Kotlin)

코틀린(Kotlin)은 Android Studio와 IntelliJ를 만든 JetBrains사에서 2010년에 출시한 프로그래밍 언어입니다. Google은 2017년 안드로이드 공식 언어로 코틀린으로 추가합니다.

자바 유료화, 그리고 자바 API 저작권에 대한 Google과 Oracle의 기나긴 소송의 여파로 보는 시각이 많습니다.

자바에 익숙한 많은 개발자들이 코틀린이라는 새로운 언어를 배우는데 부담을 느끼고 있습니다. 그러나 자바를 알고있다면 코틀린도 쉽게 배울 수 있습니다. 앞으로 코틀린의 특징과 개념에 대해 다루도록 하겠습니다.

컴파일

컴파일(Compile)은 개발자가 작성한 소스코드를 컴퓨터가 이해할 수 있는 기계어로 변환하는 과정을 의미합니다.

C의 경우 gcc라는 컴파일러가 *.c파일을 컴파일하여 *.o파일을 생성합니다. Java의 경우 자바 컴파일러(Java Compiler)가 *.java파일을 컴파일하여 자바 바이트코드를 생성합니다.

빌드

프로젝트를 진행하다보면 여러 개의 소스코드 파일이 생성됩니다. 만약 A라는 소스코드 파일에서 B라는 소스코드 파일을 참조한다고 가정해봅시다. 이때 파일 A와 파일 B를 각각 컴파일하면 A에서 B를 찾지 못하므로 참조할 수가 없습니다. 따라서 파일 A와 파일 B를 연결하는 작업이 필요한데 이를 링크(Link)라고 합니다.

빌드(Build)는 이러한 컴파일, 링크 등을 모두 포함하여 실행 가능한 소프트웨어로 만드는 과정을 의미합니다.

안드로이드 스튜디오에서 소스코드를 빌드하면 안드로이드 운영체제에서 실행할 수 있는 APK 파일이 생성됩니다.

빌드 도구

빌드 도구(Build Tool)는 소스코드를 실행 가능한 소프트웨어로 만들어주는 도구입니다. 요즘의 빌드 도구는 빌드 작업뿐만 아니라 테스트, 빌드 자동화, 배포 등 다양한 기능을 제공하기도 합니다. 자바나 코틀린의 빌드 도구에는 Ant, Maven, Gradle 등이 있습니다.
