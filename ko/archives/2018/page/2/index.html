[Kotlin] 에러 처리 (Exception & Error Handling)

2018/02/02 카테고리 Kotlin

Throwable

코틀린은 오류가 발생하면 Error나 Exception을 발생시킵니다. Error와 Exception은 모두 Throwable클래스를 상속받습니다.

Error

Error는 코드에서 잡아낼 수 없으며 프로그램 자체가 다운됩니다. 대표적으로 OutOfMemoryError, StackOveflowError가 있습니다.

Exception

Exception은 try-catch구문으로 잡아낼 수 있으며 프로그램이 다운되지 않도록 처리할 수 있습니다. 대표적으로 NullPointException이 있습니다. finally { ... }구문은 에러처리와 상관없이 무조건 실행되는 코드르입니다.

개발자가 직접 Exception을 정의하고 발생시킬 수도 있습니다. 직접 Exception을 발생시킬 때는 키워드 throw를 사용합니다.

Exception을 발생시킬 수 있는 메소드에는 @Throws 어노테이션을 붙여 에러 가능성을 표기할 수 있습니다.

[Java] Object 클래스

2018/01/29 카테고리 Java

java.lang 패키지

java.lang 패키지는 자바에서 가장 기본적인 동작을 수행하는 클래스들의 집합입니다. 따라서 자바에서는 java.lang 패키지를 import하지 않아도 바로 사용할 수 있습니다.

Object 클래스

Object클래스는 모든 자바 클래스의 최고 조상 클래스입니다. 따라서 자바의 모든 클래스는 암시적으로든 명시적으로든 Object클래스를 상속받고 있습니다. Object클래스는 java.lang패키지에 포함되며, Object클래스는 여러 메소드를 제공하고 있습니다.

equals(Object obj) 메소드

메소드를 호출하는 객체와 인자로 전달받은 객체가 동일한지 여부를 반환합니다. 다만 Object클래스를 상속받는 클래스에서는 equals(Object obj)을 직접 구현해야합니다.

getClass() 메소드

해당 객체의 클래스 타입을 반환합니다.

hashcode() 메소드

hashcode()는 객체의 해시코드를 반환합니다. 해시코드는 객체의 주소값을 해시함수에 넣어 반환된 고유한 정수값입니다. 따라서 두 객체가 동일한 객체인지 비교할 때 사용할 수 있습니다.

toString() 메소드

해당 객체의 정보를 문자열로 반환합니다.

그 외

그 외에 Object 클래스는 clone(), finalize(), notify(), notifyAll(), wait() 메소드도 제공합니다.
