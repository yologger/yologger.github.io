[Kotlin] lateinit var

2018/01/29 카테고리 Kotlin

var

코틀린에서 변수는 초기화하지 않으면 컴파일 자체가 안됩니다.

데이터 타입을 Nullable로 선언하여 null이라도 할당해주어야 합니다.

lateinit var

lateinit var는 값의 생성 및 초기화를 나중에 할 수 있도록 해줍니다. 기본 자료형에는 사용할 수 없으며 클래스의 인스턴스에만 사용 가능합니다.

문자열 String도 클래스이므로 사용할 수 있습니다.

보통 안드로이드 앱을 개발할 때 다음과 같이 사용합니다.

[Kotlin] 상수 (val, const)

2018/01/28 카테고리 Kotlin

var

변수를 선언할 때는 키워드 var를 사용합니다. 변수는 런타임에서 값을 수시로 변경할 수 있습니다.

val

val은 런타임에 값이 결정되는 상수입니다. 상수는 한 번 값이 할당되면 변경할 수 없습니다.

val은 문자열도 할당할 수 있습니다.

val는 클래스의 인스턴스도 할당할 수 있습니다.

const val

const val는 컴파일 타임에 값이 결정되는 상수입니다. 인스턴스를 할당할 수 없으며, 기본 자료형과 문자열만 할당할 수 있습니다.

클래스 안에서 사용할 때는 companion object블럭 안에서 정의합니다.

와일드 카드(Wildcard)는 제네릭 클래스의 인스턴스를 함수의 인자로 전달할 때, 그 인스턴스의 타입 변수를 제한하는 것을 의미합니다. 개념이 어려우므로 예제를 통해 살펴보겠습니다.

제네릭(Generic)이란

제네릭 (Generic)은 클래스나 메서드 내부에서 사용할 데이터의 타입을 외부에서 설정하는 것입니다. 이를 통해 컴파일 시 메서드나 클래스에 사용되는 다양한 데이터의 타입 체크를 가능하게 합니다.

제네릭 클래스

우선 제네릭(Generic)은 클래스에서 사용할 수 있습니다. 제네릭 클래스는 다음과 같이 정의합니다.

<>안에는 아직 아직 타입이 결정되지 않음을 의미하는 문자를 넣어줍니다. 이를 타입 파라미터라고 합니다. 관습적으로 대문자 E(Element), T(Type), K(Key), V(Value)를 사용합니다. 이렇게 작성된 <E>에는 어떠한 타입의 데이터도 들어올 수 있습니다.
