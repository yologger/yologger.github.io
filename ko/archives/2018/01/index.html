[자바] Object 클래스

2018/01/27 카테고리 Java

자바(Java)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 접근 제한자(Access Modifier)에 대해 알아봤습니다. 이번 포스트에서는 Object 클래스에 대해 알아보겠습니다.

java.lang 패키지

java.lang 패키지는 자바에서 가장 기본적인 동작을 수행하는 클래스들의 집합입니다. java.lang 패키지에 있는 클래스는 import하지 않아도 바로 사용할 수 있습니다.

Object 클래스

Object클래스는 모든 자바 클래스의 최고 조상 클래스입니다. 자바의 모든 클래스는 암시적으로든 명시적으로든 Object클래스를 상속받고 있습니다. Object클래스는 java.lang패키지에 포함되며, Object클래스는 여러 메소드를 제공하고 있습니다.

(1) equals()

메소드를 호출하는 객체와 인자로 전달받은 객체가 동일한지 여부를 반환합니다. 다만 Object클래스를 상속받는 클래스에서는 equals()을 직접 구현해야합니다.

다음과 같이 인스턴스를 비교할 수 있습니다.

(2) getClass()

인스턴스의 클래스 타입을 반환합니다.

(3) hashcode()

hashcode()는 객체의 해시코드를 반환합니다. 해시코드는 객체의 주소값을 해시함수에 넣어 반환된 고유한 정수값입니다. 따라서 두 객체가 동일한 객체인지 비교할 때 사용할 수 있습니다.

(4) toString() 메소드

해당 객체의 정보를 문자열로 반환합니다.

(5) 그 외

그 외에 Object 클래스는 clone(), finalize(), notify(), notifyAll(), wait() 메소드도 제공합니다.

마치며

[자바] 접근 제한자(public, private, protected, default)

2018/01/26 카테고리 Java

자바(Java)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 형 변환(Type Casting)에 대해 알아봤습니다. 이번 포스트에서는 접근 제한자(Access Modifier)에 대해 알아보겠습니다.

접근 제한자

접근 제한자는 외부에서 클래스 내부의 메소드나 멤버변수에 접근하는 것을 제한하는 것입니다. 자바는 private, protected, default, public이라는 네 개의 접근 제한자를 지원합니다. 접근 제한자는 클래스, 생성자, 메소드, 멤버변수 앞에 붙일 수 있습니다.

(1) private

private은 같은 클래스 내에서만 접근이 가능합니다. 클래스에는 붙일 수 없으며 멤버변수, 메소드, 생성자 앞에만 붙일 수 있습니다.

(2) default

접근 제한자를 생략하면 기본적으로 default 접근 제한자가 적용됩니다. default는 동일한 패키지 내에서만 접근을 허용합니다. 멤버변수, 메소드, 생성자 뿐만 아니라 클래스 앞에도 붙일 수 있습니다.

Person클래스는 com.yologger.person패키지에 정의되어있고 접근 제한자는 default입니다.

따라서 다른 패키지인 com.yologger.app에서 접근할 수 없습니다.

(3) protected

protected는 같은 패키지에서 상속 관계에 있는 클래스에서만 접근을 허용합니다. 클래스에는 붙일 수 없으며 멤버변수, 메소드, 생성자 앞에만 붙일 수 있습니다.

(4) public

public은 다른 패키지라도 어디에서든 접근이 가능합니다.

다른 패키지인 com.yologger.app에서도 접근할 수 있습니다.

마치며

이번 포스트에서는 Java의 접근 제한자(Access Modifier)에 대해 알아봤습니다. 다음 포스트에서는 Object 클래스에 대해 알아보겠습니다.

[자바] 에러 처리(Exception & Error Handling)

2018/01/21 카테고리 Java

자바(Java)에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

Throwable

자바는 오류가 발생하면 Error나 Exception을 발생시킵니다. Error와 Exception은 모두 Throwable클래스를 상속받습니다.

(1) Error

Error는 코드에서 잡아낼 수 없으며 어플리케이션 자체가 다운됩니다. 대표적으로 메모리가 부족하여 발생하는 OutOfMemoryError, StackOveflowError가 있습니다.

(2) Exception

Exception은 try-catch구문으로 잡아낼 수 있으며 프로그램이 다운되지 않도록 처리할 수 있습니다. 대표적으로 값이 null인 변수에 접근하면 발생하는 NullPointException이 있습니다.

이때 try-catch구문으로 Exception이 발생할 가능성이 있는 부분을 감싸주면 Exception이 발생해도 어플리케이션을 다운시키지 않고 에러 처리를 할 수 있습니다.

개발자가 직접 Exception을 정의하고 발생시킬 수 있습니다.

Exception을 발생시킬 가능성이 있는 함수 뒤에는 키워드throws를 사용하여 가능성을 표기할 수 있습니다.

이 함수는 try-catch구문 안에서 사용해야합니다.
