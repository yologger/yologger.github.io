[Java] 인터페이스(Interface)

2018/01/17 카테고리 Java

Java의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 Java의 추상 클래스(Abstract Class)에 대해 알아봤습니다. 이번 포스트에서는 Java의 클래스의 인터페이스(Interface)에 대해 알아보겠습니다.

인터페이스(interface)

모든 메소드가 선언만 있고 구현부가 없는 클래스를 인터페이스(Interface)라고 합니다. 인터페이스는 특정 규격을 준수하도록 강제하는데 사용합니다.

인터페이스 (Interface)를 선언할 때는 키워드 interface를 사용합니다. 인터페이스에는 메서드를 선언만 하고 구현은 하지 않습니다.

인터페이스는 인스턴스 생성할 수 없습니다.

따라서 인터페이스를 구현해야합니다. 보통 인터페이스를 구현한 클래스를 구현체(Imlementation)라고 합니다.

인터페이스를 구현할 때는 키워드 implements를 사용합니다. 인터페이스 Database 규격을 따르는 클래스 2개를 구현해보겠습니다.

위 예제에서는 인터페이스 Database라는 규격을 따르도록 강제하고 있습니다. 이렇게 하면 Database를 사용하는 입장에서 실제 구현체가 무엇인지 신경쓰지 않고 사용할 수 있습니다. 이를 어려운 용어로 다형성(Polymorphism)이라고 합니다.

위 코드 살펴봅시다. 변수 db에 MySQL의 인스턴스가 할당되든 Realm의 인스턴스가 할당되는 상관없이 save()메소드를 호출할 수 있습니다. MySql클래스와 Realm클래스가 Database인터페이스의 표준을 따르고 있기 때문입니다.

이처럼 인터페이스와 다형성을 사용하면 유지보수성을 향상시킬 수 있습니다.

마치며

이번 포스트에서는 클래스의 인터페이스(Interface)에 대해 알아봤습니다. 다음 포스트에서는 내부 클래스(Inner Class)에 대해 알아보겠습니다.

final

키워드final은 크게 3가지 용도로 사용됩니다.

1. 상수

키워드 final은 변수를 변경할 수 없는 상수로 만드는데 사용합니다.

이렇게 선언된 상수는 값을 변경할 수 없습니다.

2. 상속 불가능한 클래스

부모 클래스에 final을 붙여보겠습니다.

자식 클래스는 상속이 불가능합니다.

3. 오버라이드 불가능한 메소드

부모 클래스 메소드 앞에 final을 붙으면 오버라이드가 불가능합니다.
