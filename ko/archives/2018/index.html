[Java] 메소드 참조 (Method Reference)

2018/03/07 카테고리 Java

람다식(Lambda Expression)

람다식(Lambda Expression)이란 함수를 하나의 식(expression)으로 표현한 것입니다. 함수를 람다식으로 표현하면 메소드의 이름이 필요 없기 때문에, 람다식은 익명 함수(Anonymous Function)라고도 불립니다.

코틀린에서의 람다식

코틀린(Kotlin), 스위프트(Swift)같이 비교적 최근에 개발된 프로그래밍 언어는 자체적으로 람다식과 함수형 프로그래밍을 지원합니다. (함수형 프로그래밍은 해당 포스트에서 다루고 있습니다.)

그럼 우선 코틀린 예제를 살펴보겠습니다. 보통 코틀린에 함수는 다음과 같이 선언합니다.

코틀린은 함수를 일급객체로 취급하므로 람다식을 다음 코드와 같이 변수에 저장할 수 있습니다. (일급함수에 대해서는 이 포스트에서 확인할 수 있습니다) 언어 자체에서 함수 타입을 지원하기 때문입니다. 정수 타입에는 정수를 할당하는 것처럼 함수 타입에는 함수를 할당할 수 있습니다.

자바에서의 람다식

자바(Java)는 코틀린(Kotlin), 스위프트(Swift)같은 언어에 비해 비교적 오래되었습니다. 또한 객체지향 프로그래밍을 패러다임으로 만들어졌기에 람다식과 함수형 프로그래밍을 지원하지 않았습니다. 그러나 2014년 발표된 Java 8부터 함수형 프로그래밍을 지원하기 위한 Java API가 추가되었습니다.

아쉽게도 자바(Java)는 함수 타입이 없으므로 람다식을 변수에 저장하거나 매개변수로 전달할 수 없습니다. 이를 해결하기 위해 도입된 개념이 @FunctionalInterface어노테이션 입니다.

@FunctionInterface

함수형 인터페이스를 구현하기 위해서는 인터페이스를 정의하고 그 내부에는 1개 뿐인 abstract 함수를 선언하고, 위에는 @FunctionalInterface 어노테이션을 붙여주면 됩니다.

이제 마치 변수에 함수를 할당하는 것처럼 사용할 수 있습니다.

Java API에서는 자주 사용될 것 같은 함수형 인터페이스가 이미 정의되어있습니다.

1. Supplier

Supplier는 매개변수 없이 반환값만을 갖는 함수형 인터페이스입니다. Supplier는 T get()을 추상 메소드로 갖고 있습니다.

2. Consumer

객체 T를 매개변수로 받아서 사용하며, 반환값은 없는 함수형 인터페이스입니다. Consumer는 void accept(T t)를 추상메소드로 갖고 있습니다.

3. Function <T, R>

Function은 객체 T를 매개변수로 받아서 처리한 후 R로 반환하는 함수형 인터페이스입니다. Function은 R apply(T t)를 추상메소드로 가지고 있습니다.

4. Predicate

Predicate는 객체 T를 매개 변수로 받아 처리한 후 Boolean을 반환합니다. Supplier는 Boolean test(T t)을 추상 메소드로 갖고 있습니다.

들어가며

함수형 프로그래밍을 공부하다보면 1급 객체란 개념이 많이 언급됩니다. 이번 포스트에서는 1급 객체와 함수형 프로그래밍에 대해 알아보겠습니다.

1급 객체 (First-class citizen)

다음 3가지 조건을 모두 충족하면 1급 객체라고 할 수 있습니다.

  1. 변수나 상수에 할당할 수 있어야 한다.
  2. 함수의 인자로 전달할 수 있어야 한다.
  3. 함수의 반환값으로 반환할 수 있어야 한다.

Kotlin에서 함수는 1급 객체이지만 Java에서 함수는 1급 객체가 아닙니다. 우선 Kotlin 예제를 살펴보겠습니다.

위 예제처럼 Kotlin은 함수를 변수에 할당할 수 있으므로 함수를 1급 객체로 취급합니다. 그럼 Java 예제를 살펴볼까요?

Java는 함수를 변수에 할당할 수 없습니다. 이 외에도 Kotlin은 함수를 함수의 인자로 전달할 수 있고 함수의 반환값으로 반환할 수 있습니다. 반면 Java는 함수를 함수의 인자로 전달할 수 없고 함수의 반환값으로 반환할 수도 없습니다.

함수형 프로그래밍 언어 (Functional Programming)

함수형 프로그래밍 언어란 함수를 1급 객체로 간주하여 변수에도 할당할 수 있고 함수의 인자로 전달할 수 있으며 함수의 반환값으로 반환할 수 있는 언어를 의미합니다.

자바컴파일러와 어노테이션 프로세서

자바 컴파일러(javac)는 소스코드(.java)를 JVM(Java Virtual Machine)이 실행 가능한 바이트코드(.class)로 변환합니다. 이때 어노테이션 프로세서(Annotation Processor)를 플러그인 형태로 자바 컴파일러(javac)에 등록하여 사용합니다.

자바 컴파일러는 등록된 어노테이션 프로세서가 모두 처리될 때까지 반복하여 실행합니다. 이때 어노테이션 프로세서에 구현된 로직에 따라 경고나 에러를 발생시키기도, 추가적인 자바코드를 추가하기도 합니다. 자바 컴파일러는 이렇게 최종적으로 완성된(.java)를 바이트코드(.class)로 변환합니다.

[Java] 어노테이션(Annotation)이란

2018/03/01 카테고리 Java

들어가며

Java로 개발을 하다 보면 클래스, 메서드, 변수 앞에 @Override 같은 @ 표시를 많이 봤을 것입니다. @를 어노테이션(Annotation)이라고 합니다. 이번 포스트에서는 어노테이션에 대해 알아보겠습니다.

어노테이션(Annotation)

어노테이션은 컴파일 타임이나 런타임에 특정 처리를 하도록 정보를 제공합니다. 이를 통해 소스코드 작성 단계에서 보일러코드를 제거하고 소스코드를 간결하게 작성할 수 있습니다. 컴파일 과정에서 어노테이션을 처리하여 소스코드를 추가하거나 변경할 수 있기 때문입니다. 그 밖에도 데이터 검증 등에 활용할 수 있습니다. 어노테이션은 크게 3가지 용도로 사용됩니다.

  1. 컴파일 과정에서 에러나 경고를 감지하도록 컴파일러에게 정보를 제공
  2. 컴파일 과정에서 특정 코드나 파일을 생성/변경하도록 지시하도록 정보를 제공
  3. 런타임에서 특정 작업을 실행하도록 정보를 제공

2018/02/09 카테고리 Kotlin

제너릭 (Generic)

제네릭 (Generic)은 클래스나 메서드 내부에서 사용할 데이터의 타입을 외부에서 설정하는 것입니다. 이를 통해 컴파일 시 메서드나 클래스에 사용되는 다양한 데이터의 타입 체크를 가능하게 합니다.

클래스에서의 제너릭 (Generic)

제네릭 클래스를 정의할 땐 <>안에 임의의 타입 파라미터를 명시합니다. 보통 타입 파라미터는 관습적으로 대문자 E(Element), T(Type), K(Key), V(Value)를 사용합니다.

이렇께 정의된 제너릭 클래스는 다음과 같이 사용합니다. <>안의 타입 파리미터 T에는 어떠한 데이터 타입도 들어갈 수 있습니다.

여러 개의 타입 파라미터을 사용할 수도 있습니다.

타입 제한

키워드 where를 사용할 수도 있다.

[Kotlin] 접근 제한자 (Access Modifiers)

2018/02/08 카테고리 Kotlin

private

private으로 선언된 변수는 해당 클래스 내부에서만 접근이 가능합니다.

private으로 선언된 변수는 상속받는 자식 클래스에서도 접근이 불가능합니다.

protected

protected로 선언된 변수는 해당 클래스와 자식 클래스에서 접근이 가능합니다.

internal

internal로 선언된 변수는 같은 모듈 내에서는 어디서 접근이 가능합니다. 코틀린 공식 문서에서 말하는 같은 모듈은 아래 상황을 뜻합니다.

  1. IntelliJ IDEA Module

  2. Maven Project

  3. Gradle Source Set(with the exception that the test source set can access the internal declarations of main)

  4. a set of files compiled with one invocation of the Ant task

    쉽게 말하면 같은 프로젝트 내에서는 internal 제한자에 대해 접근 가능하다고 보면 되겠습니다.

public (default)

어디서든 접근이 가능합니다. 접근 제한자를 붙이지 않으면 기본적으로 public입니다.
