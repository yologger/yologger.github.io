[Java] 자바 플랫폼(Java Platform)의 종류

2018/04/02 카테고리 Java

Java의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

자바 플랫폼(Java Platform)에는 여러 에디션이 존재합니다. 이번 포스트에서는 대표적인 3가지 에디션의 특징에 대해 알아보겠습니다.

Java SE (Java Platform, Standard Edition)

  • 과거에는 J2SE(Java 2 SE)로 불렸습니다.
  • 가장 보편적으로 사용되는 에디션입니다.
  • 자바 데스크탑, 스마트폰 어플리케이션 개발에 사용됩니다.
  • 안드로이드 어플리케이션을 개발할 때에도 주로 Java SE가 사용됩니다.
  • 가상머신 외에도 기초적인 타입부터 네트워킹, 보안, 데이터베이스 처리, 그래픽 사용자 인터페이스 개발(Swing, AWT) 등 데스크탑 개발에 필요한 API를 포함합니다.

Java EE (Java Platform, Enterprise Edition)

  • 과거에는 J2EE(Java 2 EE)로 불렸습니다.
  • 자바 서버 애플리케이션 개발에 사용됩니다.
  • Java SE의 모든 API를 포함합니다.
  • 그 외 EJB, JSP, Sevlet, JDBC 등 서버 어플리케이션 개발을 위한 API가 포함됩니다.
  • 국내에서는 Java SE, Spring Framework, Tomcat를 조합한 기술스택에 밀려 거의 사장된 상태입니다.
  • 2017년 오라클은 오픈소스 진영인 이클립스 재단에 Java EE를 넘겼고, 명칭은 Jakarta EE로 변경되었습니다.

Java ME (Java Platform, Micro Edition)

  • Java SE API의 일부만 포함합니다.
  • 셋톱박스, PDA등 임베디드 시스템을 개발하는데 사용됩니다.


각각의 에디션도 사용하는 운영체제(Window, MacOs, Linux, ..)에 따라 다른 jdk를 다운받아야합니다. Java Platform의 모든 에디션은 이 곳에서 다운받을 수 있습니다.

[Java] 람다식(익명함수)와 함수형 인터페이스(Functional Interface)

2018/03/16 카테고리 Java

Java의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 1급 객체와 1급 함수, 그리고 함수형 프로그래밍에 대해 알아봤습니다. 이번 포스트에서는 람다식(익명함수)와 함수형 인터페이스에 대해 알아보겠습니다.

람다식(Lambda Expression)

람다식(Lambda Expression)이란 함수를 하나의 식(expression)으로 표현한 것입니다. 함수를 람다식으로 표현하면 메소드의 이름이 필요 없기 때문에, 람다식은 익명 함수(Anonymous Function)라고도 불립니다.

Java에서의 람다식

Java는 Kotlin, Swift같은 언어에 비해 비교적 오래되었습니다. 또한 객체지향 프로그래밍을 패러다임으로 만들어졌기에 람다식과 함수형 프로그래밍을 지원하지 않았습니다. 그러나 2014년 발표된 Java 8부터 함수형 프로그래밍을 지원하기 위한 Java API가 추가되었습니다.

아쉽게도 Java는 함수 타입이 없으므로 람다식을 변수에 저장하거나 매개변수로 전달할 수 없습니다. 이를 해결하기 위해 도입된 개념이 @FunctionalInterface어노테이션 입니다.

@FunctionInterface

함수형 인터페이스를 구현하기 위해서는 인터페이스를 정의하고 그 내부에는 1개 뿐인 abstract 함수를 선언하고, @FunctionalInterface 어노테이션을 붙여주면 됩니다.

이제 마치 변수에 함수를 할당하는 것처럼 사용할 수 있습니다.

Java API에서는 자주 사용될 것 같은 함수형 인터페이스가 이미 정의되어있습니다.

1. Supplier

Supplier는 매개변수 없이 반환값만을 갖는 함수형 인터페이스입니다. Supplier는 T get()을 추상 메소드로 갖고 있습니다.

2. Consumer

객체 T를 매개변수로 받아서 사용하며, 반환값은 없는 함수형 인터페이스입니다. Consumer는 void accept(T t)를 추상메소드로 갖고 있습니다.

3. Function <T, R>

Function은 객체 T를 매개변수로 받아서 처리한 후 R로 반환하는 함수형 인터페이스입니다. Function은 R apply(T t)를 추상메소드로 가지고 있습니다.

4. Predicate

Predicate는 객체 T를 매개 변수로 받아 처리한 후 Boolean을 반환합니다. Supplier는 Boolean test(T t)을 추상 메소드로 갖고 있습니다.

안드로이드에서의 람다식

람다식은 안드로이드 어플리케이션 개발에서도 자주 사용합니다. 보통 위젯을 클릭했을 때 실행할 구문을 람다식 안에 구현합니다.

우선 람다식을 사용하지 않고 익명 클래스를 사용하여 특정 버튼을 클릭했을 때 실행할 구문을 등록해보겠습니다.

[Java] 1급 객체와 1급 함수, 그리고 함수형 프로그래밍

2018/03/15 카테고리 Java

Java의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 어노테이션(Annotation)에 대해 알아봤습니다. 이번 포스트에서는 1급 객체와 1급 함수, 그리고 함수형 프로그래밍에 대해 알아보겠습니다.

1급 객체 (First-class citizen)

함수형 프로그래밍을 공부하다보면 1급 객체란 개념이 많이 언급됩니다. 다음 3가지 조건을 모두 충족하면 1급 객체라고 할 수 있습니다.

  1. 변수나 상수에 할당할 수 있어야 한다.
  2. 함수의 인자로 전달할 수 있어야 한다.
  3. 함수의 반환값으로 반환할 수 있어야 한다.

Kotlin에서 함수는 1급 객체이지만 Java에서 함수는 1급 객체가 아닙니다. 우선 Kotlin 예제를 살펴보겠습니다.

위 예제처럼 Kotlin은 함수를 변수에 할당할 수 있으므로 함수를 1급 객체로 취급합니다. 그럼 Java 예제를 살펴볼까요?

Kotlin은 함수를 함수의 인자로 전달할 수 있고 함수의 반환값으로 반환할 수 있습니다. 반면 Java는 함수를 함수의 인자로 전달할 수 없고 함수의 반환값으로 반환할 수도 없습니다.

함수형 프로그래밍 언어 (Functional Programming)

함수형 프로그래밍 언어란 함수를 1급 객체로 간주하여 변수에도 할당할 수 있고 함수의 인자로 전달할 수 있으며 함수의 반환값으로 반환할 수 있는 언어를 의미합니다.

마치며

이번 포스트에서는 1급 객체와 1급 함수, 그리고 함수형 프로그래밍에 대해 알아봤습니다. 다음 포스트에서는 람다식(익명 함수)과 함수형 인터페이스에 대해 알아보겠습니다.

어노테이션 프로세싱

어노테이션 프로세싱은 자바 컴파일러(javac)가 컴파일 단계에서 유저가 정의한 어노테이션을 분석하고 처리하는 기법입니다.

자바 컴파일러와 어노테이션 프로세서

자바 컴파일러(javac)는 소스코드(.java)를 자바 바이트코드(.class)로 변환합니다. 이때 어노테이션 프로세서(Annotation Processor)를 플러그인 형태로 자바 컴파일러(javac)에 등록하여 사용합니다.

자바 컴파일러는 등록된 어노테이션 프로세서가 모두 처리될 때까지 반복하여 실행합니다. 이때 어노테이션 프로세서에 구현된 로직에 따라 경고나 에러를 발생시키기도, 추가적인 자바코드를 추가하기도 합니다. 자바 컴파일러는 이렇게 최종적으로 완성된(.java)를 바이트코드(.class)로 변환합니다.

2018/03/02 카테고리 Java

어노테이션(Annotation)

Java로 개발을 하다 보면 클래스, 메서드, 변수 앞에 @Override 같은 @ 표시를 많이 봤을 것입니다. @를 어노테이션(Annotation)이라고 합니다.

어노테이션은 컴파일 타임이나 런타임에 특정 처리를 하도록 정보를 제공합니다. 이를 통해 소스코드 작성 단계에서 보일러코드를 제거하고 소스코드를 간결하게 작성할 수 있습니다. 컴파일 과정에서 어노테이션을 처리하여 데이터를 검증할 수 있으며, 소스코드를 추가하거나 변경할 수도 있기 때문입니다. 어노테이션은 크게 3가지 용도로 사용됩니다.

  1. 컴파일 과정에서 에러나 경고를 감지하도록 컴파일러에게 정보를 제공
  2. 컴파일 과정에서 특정 코드나 파일을 생성/변경하도록 지시하도록 정보를 제공
  3. 런타임에서 특정 작업을 실행하도록 정보를 제공

타입 소거(Type Erasure)

타입 소거(Type Erasure)는 제너릭의 타입 파라미터를 구체화된 자료형으로 대체하는 작업을 의미합니다. 자바 컴파일러는 소스 코드를 컴파일할 때 타입 소거를 수행합니다.

타입 파라미터가 Unbounded Type인지 Bounded Type인지에 따라 소거 방식이 다릅니다.

1. Unbounded Type의 타입소거

Unbounded Type은 형식 타입 파라미터로 전달받게될 실 타입 파라미터의 범위를 제한하지 않은 것을 의미합니다. <?>, <T> 형태의 타입 파라미터라고 생각하시면 됩니다.

<?>, <T>는 Object로 대체됩니다.

Unbounded Type 예제를 먼저 살펴보겠습니다.

타입 파라미터가 소거되고 Object클래스로 대치되었습니다.

2. Bounded Type의 타입소거

Bounded Type은 형식 타입 파라미터로 전달받게될 실 타입 파라미터의 범위를 제한한 것을 의미합니다. <E extends Something> 형태의 타입 파라미터라고 생각하시면 됩니다.

<E extends Something>)은 Object가 아닌 Something로 대치합니다.

Bounded Type 예제를 살펴보겠습니다.

타입 파라미터를 포함한 Observable가 Observable로 소거되었습니다.

와일드 카드(Wildcard)

와일드 카드(Wildcard)는 제네릭 클래스의 인스턴스를 함수의 인자로 전달할 때, 그 인스턴스의 타입 파라미터를 제한하는 것을 의미합니다. 개념이 어려우므로 예제를 통해 살펴보겠습니다.

super

키워드 super를 사용하면 특정 클래스와 부모 클래스만 인자로 제한할 수 있습니다.

와일드 카드<?>의 종류

  • <? extends T> 와일드 카드의 상한 제한(upper bound) - T와 그 자손들을 구현한 객체들만 타입 매개변수로 올 수 있습니다.
  • <? super T> 와일드 카드의 하한 제한(lower bound) - T와 그 조상들을 구현한 객체들만 타입 매개변수로 올 수 있습니다.
  • <?> 타입 매개변수에 제한 없습니다.
