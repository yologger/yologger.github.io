제네릭(Generic)이란

제네릭(Generic)은 클래스나 메서드 내부에서 사용할 데이터의 자료형을 외부에서 설정하는 것입니다. 제너릭을 사용하면 다음과 같은 장점이 있습니다.

  1. 특정 타입에 종속되지 않은 유연한 로직
  2. 타입 안정성
  3. 자동 형변환

장점 1) 특정 타입에 종속되지 않은 유연한 로직

제너릭을 사용하지 않았을 때의 문제점

제너릭을 사용하면 특정 타입에 종속되지 않은 유연한 로직을 구현할 수 있습니다. 우선 아래 Stack클래스 예제를 살펴봅시다.

Stack클래스는 String타입의 배열을 멤버변수로 가지고 있습니다. 따라서 다음과 같이 String타입의 데이터만 추가하고 삭제할 수 있습니다.

이제 String타입이 아닌 Integer타입의 데이터를 추가하려고 합니다. 그러나 위 Stack클래스를 사용할 수는 없습니다. Stack클래스는 String타입의 데이터만 입력받고 저장하고 삭제할 수 있기 때문입니다. 따라서 별도의 StackForInteger클래스를 정의하겠습니다.

이제 StackForInteger클래스를 사용해서 Integer타입의 데이터를 추가하고 삭제할 수 있게 되었습니다.

이제 String, Integer뿐만 아니라 더 다양한 타입의 데이터를 취급하려고 합니다. 그럼 타입이 추가될 때 마다 새로운 클래스를 정의해야 할까요? 바로 이러한 경우 제너릭을 사용할 수 있습니다.

장점 2) 타입 안정성

타입 안정성은 런타임에 발생할 에러를 컴파일 타임에 찾아낼 수 있음을 의미합니다.

오류는 컴파일 타임에서 찾아내는 것이 가장 좋습니다. 런타임에 오류가 발생하여 어플리케이션이 다운되면 비즈니스에 큰 문제가 생기기 때문입니다.

Java에서 기본적으로 제공하는 ArrayList를 쉽고 간단하게 재구현했습니다. 이제 CustomArrayList의 인스턴스를 생성하고 데이터를 넣어보겠습니다.

이제 add() 메소드에 문자열 데이터를 전달해봅시다.

형 변환이 제대로 이루어지지 않았다는 오류 메시지입니다.

컴파일 시점에 어떤 오류도 발생하지 않은 것이 가장 큰 문제입니다.

따라서 이러한 문제점을 해결하는데 제너릭을 사용할 수 있습니다.

또한 잘못된 데이터를 입력하면 컴파일 시점에 오류를 탐지하여 타입 안정성을 높여줄 수 있습니다.

장점 3) 자동 형변환

제너릭을 사용하면 자동으로 형변환을 해준다는 장점이 있습니다. 위에서 사용한 CustomArrayList클래스를 다시 살펴봅시다.

메소드 get()은 Object타입의 데이터를 반환합니다. 따라서 값을 사용하려면 형 변환을 해야합니다.

그러나 제너릭을 사용하면 별도의 형 변환이 필요없습니다.

[Java] Object 클래스

2018/01/27 카테고리 Java

Java의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 Java의 접근 제한자(Access Modifier)에 대해 알아봤습니다. 이번 포스트에서는 Java의 Object 클래스에 대해 알아보겠습니다.

java.lang 패키지

java.lang 패키지는 자바에서 가장 기본적인 동작을 수행하는 클래스들의 집합입니다. 따라서 자바에서는 java.lang 패키지를 import하지 않아도 바로 사용할 수 있습니다.

Object 클래스

Object클래스는 모든 자바 클래스의 최고 조상 클래스입니다. 따라서 자바의 모든 클래스는 암시적으로든 명시적으로든 Object클래스를 상속받고 있습니다. Object클래스는 java.lang패키지에 포함되며, Object클래스는 여러 메소드를 제공하고 있습니다.

equals(Object obj) 메소드

메소드를 호출하는 객체와 인자로 전달받은 객체가 동일한지 여부를 반환합니다. 다만 Object클래스를 상속받는 클래스에서는 equals(Object obj)을 직접 구현해야합니다.

getClass() 메소드

해당 객체의 클래스 타입을 반환합니다.

hashcode() 메소드

hashcode()는 객체의 해시코드를 반환합니다. 해시코드는 객체의 주소값을 해시함수에 넣어 반환된 고유한 정수값입니다. 따라서 두 객체가 동일한 객체인지 비교할 때 사용할 수 있습니다.

toString() 메소드

해당 객체의 정보를 문자열로 반환합니다.

그 외

그 외에 Object 클래스는 clone(), finalize(), notify(), notifyAll(), wait() 메소드도 제공합니다.

마치며

이번 포스트에서 Object 클래스에 대해 알아봤습니다. 다음 포스트에서는 제너릭(Generic)에 대해 알아보겠습니다.

[Java] 접근 제한자(public, private, protected, default)

2018/01/26 카테고리 Java

Java의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 Java의 형 변환(Type Casting)에 대해 알아봤습니다. 이번 포스트에서는 접근 제한자(Access Modifier)에 대해 알아보겠습니다.

접근 제한자 (Access Modifiers)

접근 제한자는 외부에서 클래스 내부의 메소드나 멤버변수에 접근하는 것을 제한하는 것입니다. Java는 private, protected, default, public이라는 4개의 접근 제한자를 지원합니다. 접근 제한자는 클래스, 생성자, 메소드, 멤버변수 앞에 붙일 수 있습니다.

private

private은 같은 클래스 내에서만 접근이 가능합니다. 클래스에는 붙일 수 없으며 멤버변수, 메소드, 생성자 앞에만 붙일 수 있습니다.

default

접근 제한자를 생략하면 기본적으로 default 접근 제한자가 적용됩니다. default는 동일한 package 내에서만 접근을 허용합니다. 멤버변수, 메소드, 생성자 뿐만 아니라 클래스 앞에도 붙일 수 있습니다.

Person클래스는 com.yologger.person에 정의되어있고 접근 제한자는 default입니다.

따라서 다른 패키지인 com.yologger.app에서 접근할 수 없습니다.

protected

protected는 같은 패키지에서 상속 관계에 있는 클래스에서만 접근을 허용합니다. 클래스에는 붙일 수 없으며 멤버변수, 메소드, 생성자 앞에만 붙일 수 있습니다.

public

public은 다른 패키지라도 어디에서든 접근이 가능합니다.

다른 패키지인 com.yologger.app에서도 접근할 수 있습니다.

마치며

이번 포스트에서는 Java의 접근 제한자(Access Modifier)에 대해 알아봤습니다. 다음 포스트에서는 Object 클래스에 대해 알아보겠습니다.

들어가며

Throwable

Java은 오류가 발생하면 Error나 Exception을 발생시킵니다. Error와 Exception은 모두 Throwable클래스를 상속받습니다.

Error

Error는 코드에서 잡아낼 수 없으며 어플리케이션 자체가 다운됩니다. 대표적으로 메모리가 부족하여 발생하는 OutOfMemoryError, StackOveflowError가 있습니다.

Exception

Exception은 try-catch구문으로 잡아낼 수 있으며 프로그램이 다운되지 않도록 처리할 수 있습니다. 대표적으로 값이 null인 변수에 접근하면 발생하는 NullPointException이 있습니다.

이때 try-catch구문으로 Exception이 발생할 가능성이 있는 부분을 감싸주면, Exception이 발생해도 어플리케이션을 다운시키지 않고 에러 처리를 할 수 있습니다.

개발자가 직접 Exception을 정의하고 발생시킬 수 있습니다.

직접 Exception을 발생시킬 때는 키워드throw를 사용합니다.

[Java] 익명 클래스(Anonymous Class)

2018/01/19 카테고리 Java

Java의 특징과 개념에 대해 연재하고 있습니다. 목차는 이 곳에서 확인할 수 있습니다.

들어가며

이전 포스트에서는 Java의 내부 클래스(Inner Class)에 대해 알아봤습니다. 이번 포스트에서는 Java의 클래스의 익명 클래스(Anonymous Class)에 대해 알아보겠습니다.

클래스를 상속할 때 자식 클래스를 한번만 사용하는 경우, 매번 자식 클래스를 정의하기에는 번거로운 경우가 있습니다.

이때 익명 클래스(Anonymous Class)를 유용하게 사용할 수 있습니다.

익명 클래스는 안드로이드 어플리케이션에서 이벤트를 처리할 때 많이 사용합니다.

마치며

이번 포스트에서는 익명 클래스(Anonymous Class)에 대해 알아봤습니다. 다음 포스트에서는 열거형(enum)에 대해 알아보겠습니다.
